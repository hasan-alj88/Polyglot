## Key Design Decisions (this session)

| Decision | Impact |
|----------|--------|
| `{M}` macros replace generic `<param` in `{#}` | Eliminates generic type parameter syntax entirely |
| `{M} #Name` = type macro, `{M} =W.Name` = wrapper macro | Prefix determines role; different IO markers |
| Type macros use `[#] <Param`, wrappers use `[{]`/`[}]` | Clean separation of macro kinds |
| Scalars are `##` schemas (not `#` structs) | `##Int`, `##Float` etc. are regex constraints on `#String` |
| Macro dispatch by signature (count + kind) | `<#` = type input, `<` = value input |
| 3-layer compiler bootstrap | Layer 0: hardcoded, Layer 1: no pipelines, Layer 2: full pipelines |
| Dual `{M} #Dataframe` overloads | Compile-time safe (type inputs) + runtime flexible (value inputs) |
| Map keys require `##EnumLeafs` | Predefined enum key set; arbitrary keys → use `#Serial` |
| `%This` / `%Parent` scoping | Innermost `{x}` block / one level up |
| `##CommaSeparatedList` breaks bootstrap circular dependency | Layer 1 type replaces `#Array1D:String` reference in Layer 2 |
| Namespace `=#.*` for runtime validation | Not `=Schema.*` — uses `#` prefix in `=` pipeline |
| `=UID`, `=String.Lower` | New stdlib pipelines needed by Layer 2 macros |
