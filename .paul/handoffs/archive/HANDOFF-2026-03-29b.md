## Key Design Decisions (this session)

| Decision | Impact |
|----------|--------|
| `{M}`/`{W}` split | Type macros and wrappers are separate entities with own block types |
| `[M]`/`[W]` invocation | Mirrors `{#}`/`[#]` pattern; `<~` only means inheritance |
| `[<] !Alias.Clash` fallback chain | Error-driven alias alternatives under `[#] <Alias` |
| Dataframe = Array of Map | Row-oriented; access via `$df<row<column` |
| `##` = property substitution | `[#] << ##Schema` sets `%` properties, NOT inheritance; PGE-921 for conflicts |
| `%name` / `%name.Last` | Returns definition name from `{x}` header; `.Last` splits by `.` |
| `<#type` in pipeline IO | Types as data tree inputs — extends `<#` from `{M}` to `{=}` |
| `=#.JSON/YAML/TOML` base parsers | Assumed as compiler-internal; higher-level pipelines built on top |
| `=#.Match/Validate/Describe/Coerce` | Full validation pipeline set with `<#type` inputs |
| `=#.Field` + `=File.Serial.Read.Field` | Three approaches to field extraction (tree access, safe, one-step) |
| Keep `=#list.into.Enum` in `=#.*` | Stays in `=#.*` namespace despite being compile-time |
