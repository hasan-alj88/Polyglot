## Decisions Made

| Decision | Rationale | Impact |
|----------|-----------|--------|
| Dataframe: `##` composition, not `<~` inheritance | Array constrains `<#ValueType` to `##Scalar`; `#Map` is `##Flat` — inheritance incompatible | Dataframe stands alone with shared schema properties |
| `[M]` merge = identity | Simplest mental model — outer `{#}` names result, macro fills body | Additional `[#]` lines after `[M]` extend the macro output |
| `=#.Column` for column extraction | Row-oriented Dataframe loses `$df.column` accessor | `~ForEach.Dataframe.Column` deprecated; `=#.Column` pipeline replaces it |
| Pipeline specs: IO tables not full code | User preference — simpler, clearer interface contracts | draft.md sections 1-4 rewritten as IO + error tables + descriptions |
| Keep everything under #94 | Generic removal and `{M}/{W}` split are part of #94's scope | Close #93, #79 when branch merges; #91 deferred to 94-03 |
| No new issues for `=UID`/`=String.Lower` | Part of 94-01 design, tracked in draft.md | Specs created when needed during apply |
