### Key Design Decisions Made This Session

1. **Three-tier prefix system:** `#` = type, `##` = schema (tree shape), `###` = field (leaf content)
2. **`##` schema operator:** `{#} ##Scalar`, `##Flat`, `##Deep`, `##Homogeneous`, `##Contiguous`, etc.
3. **`###` field types:** `###Value` (typed leaf) and `###Enum` (variant selector leaf)
4. **`%##`/`%###` property prefixes:** `%##Children.Uniform`, `%##Depth.Max`, etc.
5. **Leaf-only values:** Universal invariant — branches are enum/namespace only, leaves hold RawString
6. **`<` tree accessor:** `$var<key` for child access, `<` in `{#}` header for type param definition
7. **`#Dict` → `#Map`:** Renamed as base collection type
8. **`#Array` is `#Map` variant:** Keys are Cartesian product of contiguous integer enums
9. **`#Dataframe` dropped as built-in:** Row struct + `#Array` is idiomatic pattern
10. **`%##Children.Regular`:** New property — all branches at same depth have same child count (rectangular)
11. **`#IndexString`:** New string subtype for tree keys (no whitespace, `.`, `:`, `<`, `>`)
12. **No unbounded collection nesting:** Nested collections require explicit `%##Depth.Max` (PGE if missing)
13. **Schema composition:** Multiple `[#] <<` lines accumulate (one line, one expression)
14. **Redundant/contradicting schema:** PGW for redundant, PGW for override, PGE for scope conflict
