  <action>
    **Create PGW01004-orphaned-foreign-code.md** in docs/technical/compile-rules/PGW/:
    - Frontmatter: audience: developer, rule: "W1.4", code: PGW01004, name: Orphaned Foreign Code, severity: warning
    - Statement: [C] foreign code lines should only appear as children of -RT.* pipeline calls. [C] outside this scope is syntactically valid but has no effect — the compiler cannot pass the code to any runtime.
    - Rationale: The grammar accepts [C] anywhere in execution context because foreign_code_elem is a block element. Restricting [C] in the EBNF would require the grammar to understand pipeline semantics (which - prefixed names are -RT.*), which is beyond the grammar's role. A semantic warning catches misplaced [C] lines without grammar complexity.
    - Detection: compiler checks each [C] sequence; if the nearest parent [-] call is not a -RT.* pipeline, PGW01004 fires.
    - VALID example: [C] under -RT.Python.Script
    - INVALID examples: [C] at top level after -File.Text.Read, [C] inside [?] without -RT.* parent
    - See also: PGE01027 (empty foreign code block), §11.6 foreign code injection

    **Update COMPILE-RULES.md** — add PGW01004 row after PGW01003:
    `| PGW01004 | Orphaned Foreign Code |`

    **Update docs/technical/ebnf/11-control-flow.md** §11.6 — add a note after the existing Rule paragraph:
    "**Note:** PGW01004 warns when `[C]` lines appear outside the scope of a `-RT.*` pipeline call. The grammar does not restrict `[C]` placement because `foreign_code_elem` is a general block element; scoping is enforced semantically."

    Update `updated:` date to 2026-04-17 in ebnf/11-control-flow.md.
