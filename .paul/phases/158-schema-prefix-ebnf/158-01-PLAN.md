<objective>
## Goal
Add documentation to EBNF Section 05 clarifying that `[#]` serves dual roles: execution element in pipeline bodies and schema declaration element in `{#}` definition bodies.

## Purpose
Section 05 is the block element taxonomy — the reader's index for understanding what each marker does. Currently `[#]` is classified only as `execution_elem` alongside `[r]`, `[p]`, `[b]`. This is incomplete: `[#]` also introduces schema composition (`[#] << ##Scalar`), schema properties (`[#] %##Depth.Max`), and field type composition (`[#] << ###ScalarEnum`) inside `{#}` definitions. A reader or compiler implementer consulting Section 05 alone would miss these uses.

The grammar rules in Section 04 and Section 09 are already correct — this is a taxonomy/documentation gap, not a grammar bug.

## Output
- Updated `docs/technical/ebnf/05-block-elements.md` with a rule note for `[#]` dual context
</objective>
