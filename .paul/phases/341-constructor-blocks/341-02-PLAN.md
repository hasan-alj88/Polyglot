Add EBNF grammar productions for `{$}` constructor definitions and update the `%` metadata tree to include the `%definition.$` schema branch and constructor-related paths.

    **09-13-constructor.md (CREATE):**
    Follow the pattern established by 09-12-collector.md. Include:

    1. EBNF productions:
       - `constructor_def ::= "{$}" constructor_header NEWLINE constructor_body ;`
       - `constructor_header ::= "$" dotted_name '"' pattern_string '"' ;`
       - `constructor_body ::= constructor_overload+ ;`
       - `constructor_overload` containing `constructor_io`, `constructor_action`, `constructor_field_mapping`
       - `constructor_io ::= "($)" "<" capture_name ".re" "<<" '"' regex_pattern '"' ;`
       - `constructor_action ::= "[$]" "#" type_ref ;` (exactly one per overload)
       - `constructor_field_mapping ::= "[.]" "." field_name "<<" source_ref ;`
       - Native pipeline overload: `constructor_native_call ::= "[-]" pipeline_call ;` (pglib only)

    2. Rules section covering:
       - `{$}` defines a constructor (compile-time-guaranteed Final value, no error surface)
       - Constructor identifier uses `$` prefix (e.g., `$DT`, `$Path`)
       - `($)` IO lines declare regex-validated capture parameters; `.re` is mandatory
       - `[$]` type binding — exactly one per overload, after all `($)`, before `[.]`
       - `[.]` fixed field assignment maps captures to target type fields
       - Keyword overloads have no `($)` lines — entire string is exact-match regex
       - Native `[-]` calls are pglib-only; user constructors limited to string-parsing
       - Overload resolution via regex: exactly one match required; zero = error, multiple = ambiguity error on definitions
       - Structural integrity check: capture `.re` patterns must not match literal separators
       - Interpolation: `{$var}` allowed only if `$var` is constructor-sourced
       - `{$}` creates a branch on the `%` metadata tree at `%definition.$`
       - Cross-package constructors scoped by `[@]` imports

    3. Example: string-parsing overload + keyword overload (from constructors.md)

    4. Related User Documentation table mapping to constructors.md

    Use `audience: designer`, `type: spec`, `updated: 2026-04-22` frontmatter.
