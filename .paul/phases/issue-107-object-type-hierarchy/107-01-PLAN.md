<objective>
## Goal
Document the two-base-type object hierarchy in all relevant specification files: `{#}` (data) and `{=}` (pipeline) as base types, with `{!}`, `{M}`, `{Q}#Name` as `{#}` subtypes and `{T}`, `{W}`, `{Q}=Q.*` as `{=}` subtypes.

## Purpose
The type hierarchy is the foundational design decision that issues #108-#114 depend on. Without this documented, marker declarations, compiler rules, and metadata branches have no anchoring context.

## Output
Four documentation files updated with type hierarchy information, `{T}` trigger definition type, and `%T` metadata branch.
</objective>

       ```ebnf
       trigger_def         ::= "{T}" trigger_pipeline_id NEWLINE

       trigger_def_body    ::= { ( metadata_line | comment_line ) NEWLINE }

                             (* Trigger definitions contain ONLY metadata, IO declarations,
       ```

    3. Rules:

       - Must include `>IsTriggered#bool` output (mandatory). May include additional outputs.
       - Trigger identifier must use `=T.` prefix.

       ```polyglot
          [=] >NewFiles#array:path
       ```
