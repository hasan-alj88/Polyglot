<objective>
## Goal
Add a "Per-Language Encode/Decode" section to polyglot-sdk.md documenting how each supported language (Python, Rust, Go, JavaScript) converts to/from Polyglot's universal string representation.

## Purpose
The SDK spec currently defines the four public functions and type mapping table but lacks the per-language algorithms that implementors need. This section closes that gap using the brainstorm output in docs/draft.md.

## Output
- Updated `docs/technical/spec/polyglot-sdk.md` with new section containing:
  - Per-language string representation tables (Python, Rust, Go, JavaScript)
  - Complete encode/decode function signatures with code
  - "Strongly vs Weakly Typed Design Summary" table
  - Normalization rules subsection (bool, float specials, null, bytes, dt)
  - Behavior Contract → SDK type selection flow
  - Enum code-generation pattern for strongly typed languages
</objective>
