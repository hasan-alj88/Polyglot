<objective>
## Goal
Create the four individual -Run.* pipeline documentation files and seven new binding compile rules, completing the core -Run.* specification that resolves issue #281.

## Purpose
Issue #281 identified that there is no defined mechanism for binding Polyglot pipeline IO inputs to native language variable names inside [C] foreign code blocks. The Run/INDEX.md draft (already created) defines the full design: -Run.* replaces -RT.*, #Record replaces #serial for bindings, #Code:Source enum unifies inline/file code. This plan creates the per-pipeline documentation files and compile rules that make the design complete and enforceable.

- 7 new compile rule files in docs/technical/compile-rules/PGE/
- Updated Run/INDEX.md (fix PGE numbering: 01033-01039 not 01029-01035)
- Updated COMPILE-RULES.md index
</objective>

  <action>
    Create each compile rule file following the established pattern (see existing PGE01027, PGE01028 for format):

    PGE01033 — Unbound Script Variable
    - <Bind#Record field name not found as identifier in [C] code or source file
    - Only applies to <code.inline (runtime check for <code.file)

    PGE01034 — Unbound Script Output
    - >Bind#Record field name not found as identifier in [C] code or source file
    - Same inline-only validation caveat

    PGE01035 — Unbound Function Argument
    - <arg#Record field not found as parameter of <func function
    - Applies to .Function mode only

    PGE01036 — Unbound Function Kwarg
    - <kwarg#Record field not found as keyword parameter of <func function
    - Applies to .Function mode only

    PGE01037 — Bind Schema Mismatch
    - Assigned inline Record value doesn't match the declared Record schema topology
    - Applies to any <Bind, <arg, <kwarg assignment

    PGE01038 — Code Source Conflict
    - Both <code.inline and <code.file provided (violates %##Active one constraint)
    - Applies to .Function, .Script, .Bind modes

    PGE01039 — CLI Non-String Argument
    - .CLI <arg or <kwarg Record field has a type other than #string
    - CLI arguments must all be strings (shell args are always strings)

    Then update COMPILE-RULES.md to add all seven rules to the 01.x (Structure) section index.

    Each rule file must have:
    - YAML frontmatter: audience: developer, type: compile-rule, severity: error, code: PGE01NNN
    - Sections: Description, Rule (formal), Example (valid polyglot + invalid polyglot), Related
    - Cross-reference to Run/INDEX.md and relevant pipeline file
