<objective>
## Goal
Create two new specification documents defining the OTel tracing foundation for all Polyglot services: one for the tracing infrastructure itself (crate stack, span hierarchy, semantic conventions, NATS propagation) and one for user-controlled exporter configuration.

## Purpose
Issue #315 defined *what* is logged (8 permission/sandbox events) and *when*. This plan defines *how* — the shared infrastructure that all OTel-emitting components build on. Without this foundation, #315's events have no delivery mechanism, no span context, and no cross-service correlation.

## Output
- `docs/technical/spec/otel-foundation.md` — Tracing infrastructure specification
- `docs/technical/spec/otel-config.md` — Exporter and telemetry configuration specification
</objective>

    Sections to write:

    1. **Introduction** — Scope: user-controlled telemetry configuration. Users configure *where* telemetry goes without recompiling. The *what* (events) and *how* (infrastructure) are in companion specs.

    2. **Configuration Methods** — Table with 4 methods:
       | Method | Example | Priority |
       - Standard OTel env vars (OTEL_EXPORTER_OTLP_ENDPOINT, etc.) — highest priority
       - `polyglot.toml` `[telemetry]` section — default configuration
       - CLI flags (`--otel-exporter=stdout`) — override for dev/debug
       - Programmatic (internal, for testing) — lowest priority

    3. **polyglot.toml [telemetry] Section** — Full TOML schema definition:
       ```toml
       [telemetry]
       enabled = true
       exporter = "otlp"              # "otlp" | "stdout" | "none"
       endpoint = "http://localhost:4317"
       protocol = "grpc"              # "grpc" | "http"
       
       [telemetry.sampling]
       strategy = "tail-based"        # "always-on" | "ratio" | "tail-based"
       ratio = 1.0                    # For ratio strategy
       keep_errors = true             # For tail-based: always keep spans with errors/violations
       
       [telemetry.fallback]
       enabled = true
       target = "stderr"              # "stderr" | "file"
       format = "json"                # "json" only for now
       path = "/var/log/polyglot/otel-fallback.json"  # For file target
       
       [telemetry.batch]
       max_queue_size = 2048
       scheduled_delay_ms = 5000
       max_export_batch_size = 512
       ```
       Document each field: type, default, valid values, description.

    4. **Environment Variable Mapping** — Table mapping standard OTel env vars to polyglot.toml fields:
       - OTEL_EXPORTER_OTLP_ENDPOINT -> telemetry.endpoint
       - OTEL_EXPORTER_OTLP_PROTOCOL -> telemetry.protocol
       - POLYGLOT_OTEL_EXPORTER -> telemetry.exporter (Polyglot-specific)
       - OTEL_TRACES_SAMPLER -> telemetry.sampling.strategy
       Note: env vars override polyglot.toml values.

    5. **Exporter Destinations** — Brief description of supported destinations:
       - OTLP endpoint (Jaeger, Grafana Tempo, Datadog, Honeycomb) — via standard OTLP protocol
       - stdout — dev mode, human-readable
       - JSON file — for offline analysis
       - none — telemetry disabled
       Note: composable exporters (multiple destinations) are supported via config.

    6. **Fallback Logging** — Resolves open question from #315:
       - When OTel exporter is unavailable: emit structured JSON to stderr
       - Ensures violations are never silently lost
       - Fallback output includes same attributes as OTel events
       - Configurable via [telemetry.fallback] section

    7. **Sampling Strategy** — Resolves open question from #315:
       - Default: tail-based sampling
       - Tail-based keeps all spans containing errors or violations
       - Sandbox setup spans sampled at normal rate unless they contain failures
       - `keep_errors = true` ensures permission.sandbox.violation and permission.resource.kill spans are never dropped
       - Configurable via [telemetry.sampling] section

    Avoid: Implementation details (Rust code). This is a configuration specification.
    Note: This is the first document to define polyglot.toml structure. It only covers [telemetry]; other sections will be defined by future specs as needed.
