# Plan 145-01: Wait/collect markers disambiguation

**Objective:** Clarify that `[>]`/`[<]` (IO parameter handling markers) and `[*] <<`/`[*] >>` (collector IO) are distinct concepts. Rename `[>]`/`[<]` from "fallback" to "IO parameter handling." Document positional implicit IO for collectors.

## Acceptance Criteria

- AC-1: `[>]`/`[<]` described as "IO parameter handling" (not "fallback") in blocks.md and EBNF §5
- AC-2: Disambiguation note in io.md distinguishing `[*] <<`/`[*] >>` from `[>]`/`[<]`
- AC-3: io.md "Fallback IO" section renamed to "IO Parameter Handling"
- AC-4: Positional implicit IO (`<args.0`, `<args.1`...) documented for `*All`/`*First`/`*Nth` in collect.md
- AC-5: EBNF §12 documents positional implicit IO rule
- AC-6: errors.md cross-reference updated to new terminology
