## Migration Rules
The following replacements apply:

### Simple type annotation replacements (`;type` → `#type`)
- `;string` → `#string`, `;int` → `#int`, `;float` → `#float`, `;bool` → `#bool`
- `;path` → `#path`, `;serial` → `#serial`, `;RawString` → `#RawString`
- `;*` → `#*` (any-type wildcard)
- `;live` → `#live` (live metadata type prefix)

### User type replacements
- `;TypeName` → `#TypeName` (e.g., `;UserRecord` → `#UserRecord`, `;Config` → `#Config`)

### Collection type replacements (separator also changes)
- `;array.ElementType` → `#array:ElementType` (`.` → `:` for type params)
- `;array.ElementType:ND` → `#array:ElementType:ND` (dimension stays `:`)

### Field/variable/IO annotation replacements
- `.fieldName;type` → `.fieldName#type`
- `$varName;type` → `$varName#type`
- `<input;type` → `<input#type`
- `>output;type` → `>output#type`

### What NOT to replace
- `;` in EBNF production rules (terminal strings like `";"`)
- `;` at end of EBNF rules (EBNF statement terminator)
- `;` in regex patterns
- `;` in prose that isn't a type annotation
- `;` in file paths or URLs
- Files in docs/audit/, docs/draft.md, docs/technical/plan/decisions/
