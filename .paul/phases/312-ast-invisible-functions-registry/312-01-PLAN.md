<objective>
## Goal
Create a versioned AST-invisible functions registry and update all referencing docs so that PGE10014's banned constructs are sourced from the registry rather than hardcoded inline. Expand the registry with comprehensive per-language coverage researched from security literature.

## Purpose
Externalizes the banned function list into a maintainable, extensible registry — following the same pattern as `io-registry.toml`. This allows the banned list to grow without compiler code changes, supports per-language versioning, and enables packages to declare their own AST-invisible wrappers.

## Output
- New: `docs/technical/compiler/ast-invisible-registry.md` — the registry specification
- Updated: PGE10014, foreign-code-analysis algorithm, io-registry.md, user-facing foreign-code.md
</objective>

    Python — dynamic execution: eval(), exec(), compile(), code.InteractiveInterpreter.runsource(), code.InteractiveInterpreter.runcode(); dynamic loading: importlib.import_module(), __import__(), importlib.util.spec_from_file_location(), importlib.util.module_from_spec(); reflection: getattr() when used as indirect call (getattr(module, 'func')()); FFI: ctypes.CDLL(), ctypes.WinDLL(), ctypes.cdll.LoadLibrary(), ctypes.util.find_library(); serialization: pickle.loads(), pickle.load(), marshal.loads(), marshal.load()

    Rust — inline assembly: asm!() macro; dynamic loading: libloading::Library::new(), dlopen/dlsym via libc; unsafe with raw syscalls (unsafe block containing direct syscall)

    C/C++ — dynamic loading: dlopen(), dlsym(), GetProcAddress(), LoadLibrary(); inline assembly: asm, __asm__, __asm, asm volatile; process: system() (redirects to PGE10011 if {_} present)

    JavaScript — dynamic execution: eval(), new Function(), Function.prototype.constructor(string), indirect eval ((0, eval)()); dynamic loading: import(variable), require(variable); dynamic require.resolve() with variable

    Shell — dynamic execution: eval builtin; command substitution with variables: backtick `$cmd`; indirect expansion: ${!variable}

    **Each table row has columns:** Function | Category | Why Banned

    **Additional sections:**
    - Package-Level Extension (ast-invisible-registry-ext.toml format)
    - Versioning and Updates (same model as io-registry.toml)
    - Related section linking to algorithm, PGE10014, io-registry, foreign-code-parsers

    **Important scoping rules:**
    - Do NOT include process execution functions (os.system, subprocess, child_process.exec) — those are analyzable IO calls handled by the IO registry sink tables and PGE10011
    - Do NOT include general reflection (getattr for attribute access, hasattr, type()) — only getattr() used as indirect function call pattern getattr(obj, name)()
    - Do NOT include Java, Go, Ruby, PHP — only languages currently supported by the compiler (Python, Rust, C/C++, JavaScript, Shell)
