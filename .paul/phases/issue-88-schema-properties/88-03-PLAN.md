    - PGE11002: Unbounded collection nesting — collection used as value type without explicit %##Depth.Max
    - PGE11003: ### contradiction — explicit ###Value on type with untyped enum fields, or ###Enum on type with typed value fields
    - PGE11004: Invalid key type — %##Children.Type set to a type that does not inherit from #IndexString
    - PGE05005: Mixed field kinds — sibling fields mix typed (#type) and untyped (enum) declarations
    - PGE05006: ## outside {#} — ## schema reference used outside a {#} type definition

    **Warnings (PGW):**
    - PGW11001: Redundant ##/### property — schema property already inherited from parent
    - PGW11002: Contradicting ## override — schema property overrides inherited value
    - PGW11003: Unlimited depth on user type — %##Depth.Max << -1 on user-defined type

    Match the existing format and style in COMPILE-RULES.md.
  </action>

    2. **Field type reference:** `field_type = "###" identifier`
    3. **Schema composition in {#}:** `[#] << schema_type` (accumulates)
    4. **Type parameter in {#} header:** `type_param = "<" identifier` and `type_param_typed = "<" identifier "#" type_ref "<<" default_value`
    5. **Tree child accessor:** `child_access = "$" identifier "<" key ("<" key)*`
    6. **Schema property assignment:** `schema_prop = "%##" property_path "<<" value`
    7. **Field property assignment:** `field_prop = "%###" property_path "<<" value`
    8. **Type parameter constraint block:** `[<]` nested under type parameter declaration

    Integrate into the existing EBNF structure. Match the existing notation style.
    Do NOT rewrite unrelated grammar — only add/extend for schema system.
