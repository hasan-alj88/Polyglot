<objective>
## Goal
Add 6 resource limit categories (#RAM, #CPU, #GPU, #IO, #Processes, #Duration) to the {_} permission model, with #LimitAction enum for limit-exceeded behavior and cgroups v2 enforcement mapping.

## Purpose
Resource limits are permissions — permission to use a certain amount of a resource. This completes the sandbox model from #313 by specifying what resource constraints jobs operate under. Every job has resource limits (explicit via {_} or QH-defaulted), enforced via cgroups v2.

## Output
- 6 new #PermissionCategory variants with per-category capability enums
- #LimitAction enum and #LimitConfig struct types
- Resource locator fields for each category in __Permission schema
- cgroups v2 enforcement mapping in job-sandbox.md
- Limit-exceeded behavior fields in {Q} queue documentation
</objective>
