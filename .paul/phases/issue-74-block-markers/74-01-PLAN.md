<objective>
## Goal
Add three new block markers (`[+]` line continuation, `[|]` OR, `[c]` foreign code injection) to the Polyglot specification. Reassign `[+]` from OR to line continuation, migrate all OR usages to `[|]`, and document `[c]` code injection.

## Purpose
Addresses expressiveness gaps in Polyglot Code (issue #74). `[+]` line continuation enables multi-line expressions, `[|]` gives OR its own dedicated marker (avoiding overload of `[+]`), and `[c]` enables embedding foreign language code.

## Output
Updated specification in blocks.md, conditionals.md, and EBNF.md.
</objective>
