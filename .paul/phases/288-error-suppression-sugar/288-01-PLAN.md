<objective>
## Goal
Add `[!] !*-` syntax sugar for fire-and-forget error suppression. This is a one-line shorthand for `[!] !* / [-] -DoNothing` — the `-` signals "dismiss", only valid with `!*` wildcard, outputs stay Failed.

## Purpose
Reduces boilerplate for the common pattern of silently discarding errors on non-critical calls (logging, metrics, cleanup). Maintains Polyglot's exhaustive error handling principle while providing ergonomic sugar.

## Output
- EBNF grammar extended with `error_suppress` production and formalized `error_wildcard`
- User-facing error docs updated with `[!] !*-` section
- New PGW07010 compile warning rule file
- COMPILE-RULES.md index updated
</objective>

    1. Create PGW07010-suppress-on-consumed-output.md following the standard compile rule format:
       - Frontmatter: audience: developer, rule: "7.10w", code: PGW07010, name: Suppress on Consumed Output, severity: warning
       - Statement: `[!] !*-` suppresses all errors without recovery. If any output of the suppressed call is consumed downstream (wired to a variable, piped to another call), those outputs remain Failed with no replacement. PGW07010 warns that downstream code will receive Failed values.
       - Rationale: `[!] !*-` is intentional fire-and-forget at the error level. But if outputs are consumed, the developer likely expects values — the warning flags this mismatch so they can add explicit fallbacks or switch to a full `[!] !*` block with replacement pushes.
       - Detection: Compiler checks each `[!] !*-` block's parent `[-]` call. If any `(-) >output >> $target` line wires an output to a consumed variable (one that appears in a downstream expression), PGW07010 fires.
       - See also links: PGE07007 (exhaustive handling), PGW07001 (non-failable), PGW03002 (fire-and-forget [b])
       - VALID examples: !*- on call with no consumed outputs (pure side-effect like logging)
       - WARNING examples: !*- on call where >content >> $data is used downstream
       - Fix: Add (>) !> fallback on consumed outputs, or use full [!] !* block with replacement pushes
       - Diagnostic message: "Error suppression `[!] !*-` at line N under `[-]` call with consumed output `>outputName` — downstream code will receive Failed value. Add `(>) !>` fallback or use `[!] !*` with replacement push"

    2. Update COMPILE-RULES.md: Add PGW07010 row after PGW07004 in the warnings table:
       | PGW07010 | Suppress on Consumed Output |
