<objective>
## Goal
Add `%_` as a new root-level branch in the metadata tree for permissions across all three tree documentation files. Include the full permission category structure (File, Web, Database, System, Crypto, IPC, Device, Memory) and nest permissions under `%@` (package ceiling) and `%=` (pipeline-level).

## Purpose
Completes the metadata tree representation of the permission system introduced in #80 and wired into packages/pipelines in #81. Without `%_`, the tree is incomplete — permissions exist in the language but have no tree address.

## Output
- `docs/user/concepts/data-is-trees.md` — `_` Permissions in tree diagram, "How Concepts Connect" table, "Reading the Tree" prefix list
- `docs/user/concepts/metadata.md` — `%_` branch with full category/subfield listings, `._` subsections under `%@` and `%=`
- `docs/technical/spec/metadata-tree.md` — `_` in `type_prefix` rule, Object Type Branches table, and formal structure
</objective>
