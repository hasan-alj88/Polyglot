<objective>
## Goal
Resolve #Array vs #array case inconsistency by documenting that `#array` is an alias for `#Array` and standardizing usage: lowercase `#array` in type annotations, PascalCase `#Array` in prose/definition references.

## Purpose
Identifiers are case-sensitive in Polyglot. `#Array` and `#array` must have a clear, documented relationship. This follows the same convention established by #151 for scalar subtypes: `##Schema` (PascalCase schema), `#Alias` (PascalCase definition), lowercase alias in annotations.

## Output
Consistent `#Array`/`#array` usage across all docs files with the alias relationship documented.
</objective>
