<objective>
## Goal
Define the relationship between Job and Instance in Polyglot documentation. Add `{#} #Job` as a pglib struct type. Add `Instance` glossary entry. Clarify that jobs nest under pipeline instances in the metadata tree.

## Purpose
NotebookLM audit (issue #123) found that the glossary defines "Job" with UID-based identification while data-is-trees.md uses "Instance" with sequential `:N` numbering. These are different concepts but were conflated. The user has clarified:
- **Instance** = the Nth concurrent run of the same pipeline definition (`:0`, `:1`, `:2`)
- **Job** = a unit of work within a pipeline instance, created at IO boundaries, identified by UID

## Output
- Updated glossary with precise Job/Instance definitions
- New `{#} #Job` struct type in pglib
- Jobs nested under pipeline instances in metadata tree: `%=:Pipeline:N.jobs:UID`
- Cross-reference paragraph in data-is-trees.md
</objective>
