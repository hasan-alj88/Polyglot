<objective>
## Goal
Rewrite the authoritative EBNF grammar to implement the three-bracket system (`{}` define, `[]` control, `()` IO) and parallelism unification (`=` prefix for parallelism, `-` prefix for pipelines).

## Purpose
The EBNF grammar is the single source of truth for Polyglot syntax. All other documentation, compile rules, and examples derive from it. Updating EBNF first establishes the authoritative new syntax that subsequent plans will propagate.

## Output
9 EBNF files rewritten with the new three-bracket system. Every production rule, every comment, every example in the grammar reflects the new syntax.
</objective>

  <name>Task 1: Rewrite block element taxonomy and identifier prefix</name>
  <files>docs/technical/ebnf/05-block-elements.md, docs/technical/ebnf/03-identifiers.md</files>
  <action>
    **05-block-elements.md — Full taxonomy restructure:**

    The three-bracket principle creates distinct categories:

    1. Remove `data_flow_elem` category entirely — `[=]`, `[~]`, `[*]`, `[>]`, `[<]` no longer exist as block elements
    2. Add new `io_bracket` category for round-bracket markers:
       - `io_bracket ::= "(-)" | "(=)" | "(*)" | "(>)" | "(<)" ;`
       - Add rule: `(-)` = pipeline IO, `(=)` = expander IO, `(*)` = collector IO, `(>)` = output param handling, `(<)` = input param handling
    3. Update `execution_elem`:
       - `execution_elem ::= "[-]" | "[=]" | "[b]" | "[#]" ;`
       - `[-]` = sequential (was `[r]`), `[=]` = parallel (was `[p]`), `[b]` unchanged, `[#]` unchanged
    4. Remove `[-]` from `logical_elem` (reclaimed for sequential execution):
       - `logical_elem ::= "[&]" | "[|]" | "[^]" ;`
       - The `[-]` NOT comment already says "not needed" (negation uses `!` in comparison ops)
    5. Update `continuation_elem`:
       - `continuation_elem ::= "[~]" ;` (was `[+]`)
    6. Add OR control to `control_flow_elem`:
       - `control_flow_elem ::= "[?]" | "[!]" | "[T]" | "[Q]" | "[W]" | "[+]" ;`
       - `[+]` = OR scope for triggers (was `[|]`/`[⏐]` in trigger context)
    7. Update the Rule text about `[>]`/`[<]` scoping — now describes `(>)`/`(<)` under `(-)` IO lines
    8. Update the Rule text about `[#]` dual context — references to `[=]` become `(-)`

    **03-identifiers.md — Pipeline prefix change:**

    1. Change `pipeline_id ::= '=' dotted_name ;` to `pipeline_id ::= '-' dotted_name ;`
    2. Change `cross_pkg_pipeline ::= '@' name pipeline_id ;` comment from `@alias=PipelineName` to `@alias-PipelineName`
    3. Update the comment for pipeline_id: "All pipeline references use the - prefix"
  </action>
  <verify>
    Grep 05-block-elements.md for: no `[=]` in data_flow_elem, `[-]` in execution_elem, `(-)` in io_bracket, `[~]` in continuation_elem, `[+]` in control_flow_elem.
    Grep 03-identifiers.md for: `'-' dotted_name` in pipeline_id, `@alias-PipelineName`.
  </verify>
