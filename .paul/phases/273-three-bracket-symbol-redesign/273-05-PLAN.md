## Replacement Rules (from 273-01)
The 16-step replacement order (symbol reuse requires exact sequence):
1. `{=}` → `{-}` (pipeline definition block)
2. `[=]` IO marker → `(-)` (pipeline IO binding)
3. `[~]` → `(=)` (expander IO)
4. `[*]` → `(*)` (collector IO)
5. `[>]` → `(>)` (collect-into)
6. `[<]` → `(<)` (wait marker)
7. `[p]` → `[=]` (parallel invocation)
8. `[r]` → `[-]` (sequential invocation)
9. `[|]` trigger OR → `[+]` (OR trigger scope)
10. `[+]` continuation → `[~]` (line continuation)
11. `=>` → `->` (chain operator, NOT inside `>=` comparisons)
12. `~ForEach`/`~Into` → `=ForEach`/`=Into` (expander prefix)
13. Pipeline prefix `=` → `-` (e.g., `=File.Text.Read` → `-File.Text.Read`)
14. `.~` → `.=` (level iteration marker)
15. `%=` → `%-` (pipeline metadata branch)
16. `@alias=Pipeline` → `@alias-Pipeline` (import call syntax)

**CRITICAL:** Steps must be applied in order within each file to avoid double-replacement. Step 13 (pipeline `=` → `-`) must NOT touch `=ForEach`/`=Into` created by step 12.
