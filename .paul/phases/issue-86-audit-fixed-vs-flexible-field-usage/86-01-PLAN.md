# Plan: Issue #86 — Audit Fixed (.) vs Flexible (:) Field Usage

## Context

Issue #86 raised during #82 — permission categories are Polyglot-defined but used `:` (flexible) in the tree. Core rule: `.` for Polyglot-defined fixed fields, `:` for user-extensible flexible fields.

**User design decisions:**
1. **Permissions (%_)**: ALL `.` fixed — not user-extensible
2. **Errors (%!)**: Namespace names → `.` (Polyglot-defined). New `!Error` namespace — the ONLY one with `:` children (user-extensible via `{!} !Error` with `[:]` branches + `[.]` terminal leaves)
3. **Packages (%@)**: New `::` registry separator. Package names use `:` (flexible). `Registry` → `Company`. `::` appears in tree, treated as `:`
4. **General rule**: `[.]` = terminal/no more extensions. `[:]` = extensible branch. Users extend via `{x}` blocks.

### 2b. Add `!Error` namespace

New Polyglot-defined namespace — the ONLY one with user-extensible children. Users extend via `{!} !Error` using `[:]` for extensible branches and `[.]` for terminal leaves.

**Tree structure in metadata.md:**
```
├── !  (Errors)
│   ├── .File                   (fixed — Polyglot-defined)
│   │   └── .NotFound, .ReadError, .WriteError, .AccessDenied
│   ├── .No
│   │   └── .Data, .Pipeline, .Config, .Field
│   ├── .Timeout
│   │   └── .Pipeline, .Network, .Lock
│   ├── .Math
│   │   └── .DivideByZero, .Overflow, .Underflow
│   ├── .Validation
│   │   └── (defined via {!} — [.] terminal leaves)
│   ├── .Permission
│   │   └── .File.Denied, .Web.Denied, ...
│   └── .Error                  (fixed name, flexible children)
│       └── :UserDefined        (user-extensible via {!} !Error)
```

**Sibling homogeneity rule:** All children of a parent must use the same separator — either all `[:]` or all `[.]`. No mixing at the same level. Either separator can appear at any depth (not just leaves).

**User code example:**
```polyglot
{!} !Error
   [:] :MyApp
      [:] :Auth
         [.] .Expired;#Error
         [.] .Invalid;#Error
      [:] :GeneralFailure;#Error
```

`[:] :Auth` and `[:] :GeneralFailure` are siblings → both `[:]`. Under `:Auth`, `[.] .Expired` and `[.] .Invalid` are siblings → both `[.]`.

## Task 3: Package Address — `::` Separator + `Company` Rename

### 3a. New address format

**Before:** `@Local:999.MyPkg.Sub:v1.2.3.2`
**After:** `@Local:999::MyPkg:Sub:v1.0.0`

Changes:
- `::` separates registry+ID from package name
- Package name segments use `:` (flexible, user-defined)
- `Registry` renamed to `Company`
- Community ID: `devops.NotificationHub` → `devops:NotificationHub` (all `:`)
- Company domain: `acme.corp` → `acme:corp` (all `:`, user-extensible)
- `::` in tree paths treated as `:`

**New registry table:**

| Registry | ID Format | Full Example |
|----------|-----------|--------------|
| `Local` | `:<port>` | `@Local:999::MyPkg:v1.0.0` |
| `Community` | `:<username>:<project>` | `@Community:user123:ProjectName::MyPkg:v1.0.0` |
| `Company` | `:<domain parts>` | `@Company:acme:corp::MyPkg:v1.0.0` |

**New address structure:**
```
@<Registry>:<ID>::<PackageName>:<Version>
 ├── Registry   — fixed (Local/Community/Company)
 ├── :ID        — flexible (format per registry type)
 ├── ::         — registry separator
 ├── :PkgName   — flexible (user-defined, can nest: :MyPkg:Sub)
 └── :Version   — flexible (semver)
```

## Task 4: Path Grammar Update

`docs/technical/spec/metadata-tree.md` lines 20-31 — update EBNF and table:

**Current grammar:**
```ebnf
schema_path   ::= "%" "definition" "." type_prefix ":" ref
instance_path ::= "%" type_prefix ":" ref ":" instance { "." field }
```

**Updated — add branch-specific patterns:**
```ebnf
schema_path     ::= "%" "definition" "." type_prefix ":" ref
instance_path   ::= "%" type_prefix ":" ref ":" instance { "." field }
permission_path ::= "%" "_" { "." field }
error_path      ::= "%" "!" "." namespace { "." leaf }
                   | "%" "!" ".Error" { ":" user_path }
package_path    ::= "%" "@" ":" registry { ":" id_part } "::" name { ":" segment }
```

Update the element table to note:
- `%_` uses only `.` (no ref/instance levels)
- `%!` uses `.` for namespace and stdlib paths, `:` under `.Error`
- `%@` uses `::` separator
