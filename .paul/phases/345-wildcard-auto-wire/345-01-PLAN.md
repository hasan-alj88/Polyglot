<objective>
## Goal
Define the `<* << $A>*` wildcard auto-wire syntax: EBNF grammar productions, user documentation, updated compile rules (rewritten from retired chain context to labeled `[-]` call context), edge cases, and glossary entry.

## Purpose
Issue #340 retired the `->` chain operator. The one convenience lost was auto-wiring between adjacent steps. Issue #345 recovers this as a general-purpose feature using `<* << $A>*` — wildcard IO wiring with compile-time bijective type-topology matching. Four existing compile rules (PGE08001-08003, PGW08001) still reference the retired chain syntax and must be rewritten.

## Output
- EBNF productions for `wildcard_io_source`, `wildcard_io_target` in §7 and updated `call_io_line` in §10
- New user doc `auto-wire.md` explaining the feature with examples
- Rewritten PGE08001-08003 and PGW08001 using `<* << $A>*` syntax (not chain `->` syntax)
- Edge cases for wildcard auto-wire (valid and invalid)
- Glossary entry for "wildcard auto-wire"
</objective>

    **EBNF §7 — IO Parameters:**
    Add after label_access production:
    - `wildcard_output ::= variable_id ">*" ;` — all outputs of a labeled operation
    - `wildcard_input  ::= "<*" ;` — all inputs of the current pipeline call
    Add a prose rule: `<* << $Label>*` requires bijective type-topology matching. Compiler matches outputs to inputs by TYPE-IDENTITY. Match must be bijective and onto — every output maps to exactly one input, every input receives exactly one output. If match fails, developer must use explicit per-port wiring.

    **EBNF §10 — Execution:**
    Add wildcard auto-wire as a valid `call_io_line` alternative:
    - `call_io_line ::= ... | "(-)" wildcard_input assignment_op wildcard_output ;`
    Add a prose rule: `<* << $A>*` requires ALL of A's outputs to be Final before the target pipeline triggers. This naturally produces completion-wait behavior.

    **Compile Rules — Rewrite all four:**
    - PGE08001: Rewrite Statement to describe wildcard auto-wire type mismatch (not chain). Replace example with labeled [-] calls using <* << $Label>*. Detection: compiler attempts 1-to-1 type matching via TYPE-IDENTITY; fires when an output type has no corresponding input type.
    - PGE08002: Rewrite for wildcard context. Example: two outputs share #string type, one input accepts #string — ambiguous which maps where. Detection: after type pairing, duplicate types on either side.
    - PGE08003: Rewrite as port count mismatch (issue says bijective and onto). Example: $A has 3 outputs, target has 2 inputs — count mismatch. Detection: output count != input count.
    - PGW08001: Rewrite for wildcard context. Auto-wire succeeded but explicit wiring preferred for clarity. Example: single-output-to-single-input wildcard succeeds with warning.

    Remove all references to -> chain operator, step numbering (>0., <1.), and chain_call from these files. Update "See Also" sections to reference auto-wire.md instead of chains.md.
