Expand the existing "Normalization Rules" subsection in polyglot-sdk.md into a full "Wire Format Conventions" section that explicitly documents canonical float/null/boolean wire format conventions with schema dependencies.

  <action>
    Replace the existing "### Normalization Rules" subsection (lines 823-849) with a more comprehensive "### Wire Format Conventions" section containing:

    1. **Intro paragraph** — The SDK normalizes all values to canonical wire format strings before serialization. This section documents the conventions that apply across all supported languages.

    2. **#### Boolean** subsection:
       - Canonical wire format: `"True"` / `"False"` (Python capitalization)
       - Per-language normalization table:
         | Language | Native Form | SDK Normalizes To |
         | Python | `True`/`False` | `"True"`/`"False"` (no change) |
         | Rust | `true`/`false` | `"True"`/`"False"` |
         | Go | `true`/`false` | `"True"`/`"False"` |
         | JavaScript | `true`/`false` | `"True"`/`"False"` |
       - Cross-ref: per [[native-dispatch#Value Encoding]]

    3. **#### Float Special Values** subsection:
       - Governed by schema properties — not all floats support these values
       - `##Nullable` enables `""` (empty string = none)
       - `##Inf` enables `"Infinity"`, `"-Infinity"`, `"NaN"`
       - Per-language normalization table:
         | Value | Canonical | Python | Rust | Go | JavaScript |
         | +Infinity | `"Infinity"` | `"inf"` | `"inf"` | `"+Inf"` | `"Infinity"` |
         | -Infinity | `"-Infinity"` | `"-inf"` | `"-inf"` | `"-Inf"` | `"-Infinity"` |
         | NaN | `"NaN"` | `"nan"` | `"NaN"` | `"NaN"` | `"NaN"` |
       - SDK normalizes on write (to_polyglot), accepts all variants on read (from_polyglot)
       - Cross-ref: [[jm3lib/types/schemas/Inf|##Inf]], [[jm3lib/types/schemas/Nullable|##Nullable]]

    4. **#### Null/None** subsection:
       - Wire format: empty string `""` with type `"none"`
       - Consistent with ###None field type semantics
       - Per-language mapping:
         | Language | Native | Wire |
         | Python | `None` | `""` (type: `"none"`) |
         | Rust | `Option::None` | `""` (type: `"none"`) |
         | Go | `nil` | `""` (type: `"none"`) |
         | JavaScript | `null` | `""` (type: `"none"`) |

    5. Keep existing Bytes, DateTime, and JavaScript integer overflow paragraphs after the new subsections.

    Add cross-reference comments at top of file if not already present:
    - `<!-- @c:jm3lib/types/schemas/Inf -->`
    - `<!-- @c:jm3lib/types/schemas/Nullable -->`

    Avoid: changing any other section of polyglot-sdk.md. Do not modify the Per-Language Encode/Decode section or the Primitive Type Mapping Table.
