<task type="auto">
  <name>Task 2: Define 4 constructors ($Ver, $URL, $IP, $Color) and 4 parse pipelines</name>
  <files>docs/user/pglib/constructors/Ver.md, docs/user/pglib/constructors/URL.md, docs/user/pglib/constructors/IP.md, docs/user/pglib/constructors/Color.md, docs/user/pglib/pipelines/Ver.Parse.md, docs/user/pglib/pipelines/URL.Parse.md, docs/user/pglib/pipelines/IP.Parse.md, docs/user/pglib/pipelines/Color.Parse.md</files>
  <action>
    **$Ver constructor** (basic string-parsing pattern):
    - Full semver: `$Ver"{major}.{minor}.{patch}"` — captures map to .major, .minor, .patch
    - With prerelease: `$Ver"{major}.{minor}.{patch}-{pre}"` — adds .prerelease
    - With build: `$Ver"{major}.{minor}.{patch}+{build}"` — adds .build
    - With both: `$Ver"{major}.{minor}.{patch}-{pre}+{build}"`
    - ($) .re constraints: major/minor/patch = `[0-9]+`, pre = `[a-zA-Z0-9.]+`, build = `[a-zA-Z0-9.]+`
    - 4 overloads, resolution by presence of `-` and `+` separators
    
    **$URL constructor** (native pipeline conversion pattern, like $Dur):
    - Full URL: `$URL"{url}"` — single permissive capture, [-] -URL.Decompose native pipeline decomposes into fields
    - ($) .re << `.+` (permissive), native PgUrl class validates RFC 3986 structure
    - Uses native validation (like $Re) + native pipeline conversion (like $Dur) — hybrid pattern
    - [-] -URL.Decompose produces >scheme, >host, >port, >path, >query, >fragment
    
    **$IP constructor** (native validation pattern, like $Re):
    - IPv4: `$IP"{addr}"` with ($) .re << `[0-9]+\.[0-9]+\.[0-9]+\.[0-9]+`
    - IPv6: `$IP"{addr}"` with ($) .re << `.+` (containing `:`)
    - PgIP native class validates ranges and format at compile time
    - 2 overloads: distinguished by `:` presence (IPv6) vs `.` only (IPv4)
    - Single .address field populated from <addr
    
    **$Color constructor** (mixed pattern):
    - Hex 6-digit: `$Color"#{hex}"` — ($) .re << `[0-9a-fA-F]{6}`, [-] -Color.HexToRGB decomposes
    - Hex 8-digit (with alpha): `$Color"#{hex}"` — ($) .re << `[0-9a-fA-F]{8}`, [-] -Color.HexToRGBA
    - Named: `$Color"{name}"` — ($) .re << `[a-z]+`, PgColor native class validates name + looks up RGB values
    - 3 overloads: hex with `#` prefix (6 or 8 chars), named (no `#` prefix)
    - Native pipelines convert hex→RGBA fields
    
    **Parse pipelines** (all follow Re.Parse pattern):
    - -Ver.Parse: <raw#string → >ver#ver, !Parse.Ver.InvalidFormat
    - -URL.Parse: <raw#string → >url#url, !Parse.URL.InvalidFormat / !Parse.URL.InvalidScheme
    - -IP.Parse: <raw#string → >ip#ip, !Parse.IP.InvalidFormat / !Parse.IP.OutOfRange
    - -Color.Parse: <raw#string → >color#color, !Parse.Color.InvalidFormat / !Parse.Color.UnknownName
    
    Each constructor doc follows the established structure: frontmatter, cross-refs, overload code blocks, overload resolution table, usage examples (constructor vs parse pipeline).
    Each parse pipeline doc follows the Re.Parse.md pattern: {N} native definition, IO table, Error table, Permissions, Usage example, Metadata, Related.
  </action>
  <verify>All 8 files exist with correct {$} blocks showing ($) regex, [$] type binding, [.] field mapping; all parse pipelines have {N} definition, IO, errors</verify>
