<objective>
## Goal
Document the `-T.Git.*` trigger family and `#Git` type tree for CI/CD pipelines. Create pglib trigger docs, the `#Git` data type definition, and update existing docs to reference the new triggers.

## Purpose
CI/CD is the most natural use case for a trigger-driven pipeline language. Git triggers let Polyglot serve as a native CI/CD platform, competing with GitHub Actions and GitLab CI while staying true to its trigger-driven architecture. Approved draft content is in `docs/draft.md`.

## Output
- 4 new trigger pipeline docs (`Git.Hook.md`, `Git.Push.md`, `Git.PR.md`, `Git.Tag.md`)
- 1 new type doc (`git.md`) for the `#Git` type tree
- Updated `T/INDEX.md` with new triggers
- Updated `io-triggers.md` with Git trigger examples
- Updated `technical-ops.md` DevOps scenarios to reference Git triggers
</objective>

    Create 5 new files based on the approved draft in docs/draft.md:

    **git.md** — #Git type tree documentation:
    - Follow pattern from FileAccess.md for structure
    - Use the single {#} #Git definition from draft (nested subtypes)
    - Document all enum subtypes with ##Enum schema properties
    - Document all struct subtypes with field tables
    - Include metadata paths for definition and instances

    **Git.Hook.md** — -T.Git.Hook trigger:
    - {N} native def with #NativeKind.Trigger
    - %InlineString for hook name inline arg: -T.Git.Hook"post-commit"
    - Input: hook name (via inline string)
    - Outputs: >sha#string, >files#array.Git.DiffStat
    - Permission: System.Process
    - Design note: thin shell dispatcher POSTs to localhost, same HTTP path as -T.Webhook

    **Git.Push.md** — -T.Git.Push trigger:
    - {N} native def, semantic trigger (works over local hook OR remote webhook)
    - Filter inputs: branch patterns, path globs
    - Outputs: >branch#string, >sha#string, >commits#array.Git.Commit
    - Permission: System.Process (local) or Web.Socket (remote)

    **Git.PR.md** — -T.Git.PR trigger:
    - {N} native def, remote-only semantic trigger
    - Filter inputs: action enum (#Git.PRAction), target branch
    - Outputs: >number#int, >title#string, >source#string, >target#string, >sha#string, >commits#array.Git.Commit
    - Permission: Web.Socket

    **Git.Tag.md** — -T.Git.Tag trigger:
    - {N} native def, semantic trigger (local or remote)
    - Filter input: tag name pattern (glob)
    - Outputs: >name#string, >sha#string, >tagger#Git.Author
    - Permission: System.Process (local) or Web.Socket (remote)

    For all trigger docs, follow the exact template from Webhook.md and Folder.NewFiles.md.
    Include the three-tier model note (transport vs semantic) from the draft design notes.
  </action>
