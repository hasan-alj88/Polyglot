## Goal
Create `docs/philosophy/symbology.md` documenting the visual design rationale behind Polyglot's prefix characters, bracket system, and assignment direction.

## Purpose
Polyglot's syntax is deliberately symbolic — every prefix, bracket type, and assignment operator was chosen for visual meaning. This rationale is scattered across the developer's head and informal discussions but never collected into a single authoritative document. The philosophy/ folder (created by #331) is the right home for this content.

       - Markers declare which section each line belongs to
       - No ambiguity about line purpose

       - (X) IO — IO declarations, collector IO, direction
       - Reference [[syntax/blocks]] with @u:.

       - << / >> final (immutable), <~ / ~> default (overwritable once)

       - Push/pull affects pipeline design
