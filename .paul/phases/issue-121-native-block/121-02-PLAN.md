    Rewrite PGE01028 — the core rule changes from ".baseCode vs body" to "{N} vs {=} block type":
    - Rename: "Native/Derived Block Mutual Exclusion" (was "Base/Derived Pipeline Mutual Exclusion")
    - Statement: {N} blocks are native (no execution body, must have %Native.Kind); {=} blocks are derived (must have execution body, cannot have %Native metadata)
    - Update sub-conditions:
      (a) {N} with execution body → error
      (b) {=} with %Native.Kind → error (derived cannot be native)
      (c) {N} without %Native.Kind → error (native must declare kind)
      (d) %Native.Kind references invalid #NativeKind variant → error
      (e) %Native.<Language> references unknown language → error
    - Update all examples to use {N} block syntax:
      VALID: {N} =File.Text.Read with [%] .Kind << #NativeKind.Execution
      VALID: {=} =ProcessData with [T]/[Q]/[W]/body
      INVALID: {N} with [r] body, {=} with %Native.Kind, etc.
    - Keep {T}/{Q} IO-only exception (derived triggers/queues can be bodyless)
  </action>
  <verify>PGE01028 references {N} not .baseCode; sub-conditions use #NativeKind; examples use {N} block</verify>
