<task type="auto">
  <name>Task 1: Convert DT.md pipeline definitions to {N}</name>
  <files>docs/user/stdlib/pipelines/DT.md</files>
  <action>
    For each of the ~40 pipeline definitions in DT.md:
    1. Replace `{=} =DT.*` with `{N} =DT.*`
    2. Remove the `[ ] Base pipeline — compiler intrinsic` comment line
    3. Add after the {N} line: `[%] .Kind << #NativeKind.Execution`
    4. Add: `[%] .Rust << "<CamelCaseName>"` (derive from pipeline name, e.g., =DT.Now → "DtNow", =DT.From.Epoch → "DtFromEpoch")
    5. Keep existing [%] .description, [%] .version, [=] IO, and [=] ! error lines unchanged
    
    Also update intro text:
    - "All `=DT.*` pipelines are base pipelines (compiler intrinsics)" → "All `=DT.*` pipelines are native definitions (`{N}` blocks)"
    
    Pattern for each definition (before → after):
    ```
    BEFORE:
    {=} =DT.Now
       [ ] Base pipeline — compiler intrinsic
       [%] .description << "..."
       [%] .version << "1.0.0"
       [=] >now#dt
    
    AFTER:
    {N} =DT.Now
       [%] .Kind << #NativeKind.Execution
       [%] .Rust << "DtNow"
       [%] .description << "..."
       [%] .version << "1.0.0"
       [=] >now#dt
    ```
  </action>
  <verify>grep -c "{=}" DT.md returns 0 for pipeline defs; grep -c "{N}" returns ~40; no "Base pipeline" comments remain</verify>
  <done>AC-1: All DT.md pipeline definitions use {N} with %Native metadata</done>
</task>
