# Plan 92-02: Resolve Datatype Edge Cases

## Objective

Review each EC-24.x edge case from 92-01's audit and apply fixes: regex corrections, new compile rules (PGE/PGW), design decisions, and spec updates.

## Tasks

### Task 1: Review all 18 edge cases one by one
Walk through EC-24.1 through EC-24.18, determine status (clean / fix needed / TBD).

### Task 2: Fix #Dimension regex (EC-24.3)
Correct regex from `"^[0-9]+$D"` to `"^[0-9]+D$"` in syntax/types.md and aj3lib/types/scalars.md.

### Task 3: Add PGE11005 — Final Field Override via Inheritance (EC-24.7)
Add compile rule preventing child types from overriding `<<` final fields via `<~` inheritance.

### Task 4: Add ###None field type and PGE04021 (EC-24.10)
Add `###None` as third field type for nullable types. Only `###None` types accept empty string `""`. Add PGE04021.

### Task 5: Document 0D array semantics (EC-24.13)
0D array = scalar container, direct access, no indexing. PGE04017 on index attempt.

### Task 6: Update EDGE-CASES.md
Mark resolved edge cases with Status lines, update code blocks and follow-up issues.
