<objective>
## Goal
Resolve two EBNF edge cases from issue #307: document that mixed separators in `field_path` are valid across level boundaries (X.41), and remove redundant `cross_pkg_enum` from `match_value` production (X.42).

## Purpose
Tighten the EBNF grammar where redundant, and explicitly document intentional permissiveness where it exists, so the spec is unambiguous for compiler implementers.

## Output
- EBNF §3.2 `field_path` gains explicit comment about cross-level mixed separators
- EBNF §11.1.1 `match_value` loses redundant `cross_pkg_enum` alternative
- Two new edge cases (EC-3.8, EC-11.10) documenting these decisions
- PGE05001 gains clarifying note about per-level scope vs per-path scope
</objective>
