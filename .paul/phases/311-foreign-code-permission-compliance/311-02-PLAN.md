    2. **Phase 1: Capability Map Construction** — extract from pipeline's (-) _PermName references

    4. **Phase 3: Resource Argument Extraction** — 5 forms table (literal, variable, concatenation, f-string, unresolvable) with confidence levels
    5. **trace_assignment** — intraprocedural constant propagation pseudocode, same-function-only rule, <Bind parameter resolution
    6. **Scope Matching** — file (normalize + glob), network (host + port), DB (connection string parsing), with examples
    7. **Bottom-Up IO Tracing** — focus on lowest-level IO calls; functions not in sink table but containing known IO subcalls are flagged accordingly
    8. **Worked Examples** — at least the violation detection and variable tracing examples from brainstorm

    Frontmatter: audience: developer, type: specification, status: complete

    1. **Overview** — separate config file (io-registry.toml), versioned independently of compiler

    6. **Known-Pure Functions** — builtins per language that are silently skipped (len, str, sorted, etc.)
    7. **Deferred Categories** — table of System/Crypto/IPC/Device/Memory with examples and priority
    8. **Package-Level Extension** — how developers add entries for packages not in the registry
    9. **Versioning and Updates** — ships with compiler, registry patches, community submission model

    Frontmatter: audience: developer, type: specification, status: complete
    Cross-references: @c:technical/algorithms/foreign-code-analysis, @u:concepts/permissions/foreign-code, PGW10005

    **compliance-report.md:**
    1. **Overview** — compiler generates a compliance report as part of the Behavior Contract
    2. **Report Structure** — per-pipeline: pipeline name, declared {_} permissions, analyzed IO calls (function, category, resource, verdict), warnings list
    3. **Verdict Types** — PASS (within scope), FAIL (PGE10013), UNVERIFIABLE (PGW10002), UNKNOWN (PGW10005), OPAQUE (PGW10003)
    4. **Privacy Considerations** — no source code in reports (only call signatures + verdicts), no credential values, path patterns only
    5. **Integration** — emitted alongside the Behavior Contract; sandbox restrictions derived from the same data
    6. **Example Report** — show a sample compliance report for a pipeline with mixed verdicts

    Frontmatter: audience: developer, type: specification, status: complete
    Cross-references: @c:technical/spec/behavior-contract, @u:concepts/permissions/enforcement, @c:technical/algorithms/foreign-code-analysis

  <files>docs/technical/compiler/foreign-code-parsers.md</files>
  <action>

    2. **tree-sitter (Universal Base)** — table of grammar crates per language with AST quality and notes (from brainstorm)

    7. **Integration Points** — how the parser output feeds into the detection algorithm (foreign-code-analysis.md)
