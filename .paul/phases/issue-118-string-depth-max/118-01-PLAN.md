<objective>
## Goal
Redefine the `##Scalar` schema, introduce `##Leaf`, and add `###ScalarValue`/`###ScalarEnum` leaf types in the four core reference files that define and explain these concepts.

## Purpose
Issue #118: `#String` has `##Scalar` (Depth.Max=0, "no flexible children") but the metadata tree shows `:int`, `:float` as flexible children under `%#:String`. The resolution: fixed fields count toward depth, so `##Scalar` should be Depth.Max=1, and `##Leaf` (Depth.Max=0) is reserved for truly atomic types like `RawString`. Additionally, `##Scalar` is specifically for the `#String:*` family and scalar enums, constrained to `###ScalarValue` or `###ScalarEnum` leaf types.

## Output
Updated core reference docs with new schema/leaf type definitions. Plan 118-02 will propagate these changes across all remaining docs.
</objective>
