<objective>
## Goal
Add the `-Run.Shell` pipeline specification and the `System.Shell` permission capability. This pipeline executes shell command strings through the system shell (`sh -c "..."`), enabling piped commands, redirections, and compound shell expressions that `-Run.<Lang>.CLI` cannot handle.

## Purpose
Many automation scenarios (git commands, docker builds, kubectl, piped shell expressions) require shell interpretation. `-T.Git.*` triggers (#284) depend on this capability for local hook activation. `-Run.Shell` fills the gap between structured binary invocation (CLI) and arbitrary shell execution.
