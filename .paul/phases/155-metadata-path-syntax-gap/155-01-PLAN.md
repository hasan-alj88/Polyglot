<objective>
## Goal
Add an "Advanced Metadata Paths" section to metadata.md that bridges the simple shorthand accessors (`$name%state`) with the full formal path syntax (`%$:myVar:<current>.state`) and documents instance addressing.

## Purpose
Users learn simple accessors but have no path to understanding instance-level metadata access or the full grammar. The technical spec (path-grammar.md) already defines this but is audience-inappropriate for users.

## Output
- Updated `docs/user/concepts/metadata.md` with advanced metadata path section
</objective>

    Add a new section before the "## Rules" section (after the Native {N} live fields section) titled "## Advanced: Full Metadata Paths".

    Content should include:
    1. Brief explanation that shorthand accessors resolve to full tree paths at compile time
    2. Table showing shorthand → full path resolution (draw from path-grammar.md lines 49-56):
       - `=MyPipeline%status` → `%=:MyPipeline:<current>.status`
       - `$myVar%state` → `%$:myVar:<current>.state`
       - `#Record%lastModified` → `%#:Record:<current>.lastModified`
    3. Explain `:<current>` — implicit, resolves to calling context's instance
    4. Instance addressing — when a pipeline has multiple instances, use `:N` to target a specific one:
       - `%=:MyPipeline:3.status` (instance 3's status)
       - Note: most code uses `:<current>` implicitly; explicit `:N` is for monitoring/debugging
    5. Cross-reference: "For the complete path grammar, see [[metadata-tree/path-grammar|Path Grammar]]."
