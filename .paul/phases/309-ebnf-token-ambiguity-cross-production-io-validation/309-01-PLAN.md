    1. In docs/technical/ebnf/06-operators.md §6.3, after the range_expr production and before the `---` separator, add a **Lexer Disambiguation** rule note:
       - `[?]` is a three-character block element token, matched at line start after indentation
       - `?[` and `?(` are two-character range tokens, matched in expression context after a `value_expr`
       - Comparison operators consume `?` greedily: `=?[` tokenizes as `=?` + `[`, never `=` + `?[`
       - These rules are positional (line-start vs mid-expression) — no grammar ambiguity exists

    2. In docs/technical/edge-cases/06-operators.md, append EC-6.6 after EC-6.5:
       - Title: `EC-6.6: Range token ?[ vs conditional [?] (X.45)`
       - EBNF ref: range_open §6.3, control_flow_elem §5.1
       - Decision: Accept — lexer context disambiguates
       - Three code examples:
         a. Range in assignment: `[-] $ok#bool << $score ?[60, 100]` — `?[` is mid-expression
         b. Conditional: `[?] $score >=? 60` — `[?]` is line-start block element
         c. Greedy parsing: `[?] $x =? $y` — `=?` consumed as one comparison operator, `?` not available for `?[`
       - Update the `updated:` date in frontmatter
  </action>
