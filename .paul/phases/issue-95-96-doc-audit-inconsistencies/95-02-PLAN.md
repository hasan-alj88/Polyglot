## Key Conversion Rules
Each `{#}` type definition MUST have:
1. `[%] .description << "..."` and `[%] .version << "1.0.0"` metadata
2. `[#] << ##Scalar` (for flat types) or omit ## for multi-level structs
3. `[#] << ###Enum` (enum fields ‚Äî no #type) or `[#] << ###Value` (typed value fields)
4. `[#] %##Alias << "..."` where applicable
5. `[.]` for fixed fields, `[:]` for flexible fields ‚Äî never mixed at same sibling level (PGE05001)
6. Enum fields: no `#type` annotation. Value fields: must have `#type`.
7. 3-space indentation throughout

## #WeekSystem Fix (Option A)
`#WeekSystem` brainstorming has `.ISO#BusinessWeek` ‚Äî enum field with type annotation violates PGE05005.
Fix: Make `#WeekSystem` a struct with enum fields, each with a nested `.config#BusinessWeek` value sub-field.
Follows the `#Severity` pattern from structs.md (enum field + nested value sub-fields).

- `datetime-calendars.md` ‚Äî calendar-specific date types + calendar months
