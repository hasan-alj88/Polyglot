## Pipeline Conversion Rules
Each `{=}` pipeline definition MUST have:
1. `[%] .description << "..."` and `[%] .version << "1.0.0"` metadata
2. `[T]` trigger with IO declarations (`[=] <input`, `[=] >output`)
3. `[Q]` queue configuration
4. `[W]` wrapper declaration
5. `[_]` permission declaration — most =DT.* pipelines need `_None` (pure computation)
6. Base pipeline marker if applicable
7. 3-space indentation

## Permission Assessment
- =DT.Now → `[_] _IO.Read` (reads system clock)
- All other =DT.* → `[_] _None` (pure computation — no IO, no network, no filesystem)
