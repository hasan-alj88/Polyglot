    Create three compile rule docs following the established format from existing PGE/PGW files (use PGE10014 as pattern reference).

    **PGE10015 — Opaque Binary Without _Unsafe.SandboxOnly:**
    - Severity: Error
    - Category: Permission / Foreign Code Safety
    - Trigger: A -Run.*.CLI pipeline or any pipeline where the compiler cannot perform full AST analysis on the foreign code (compiled binary, no tree-sitter support)
    - Message: "Opaque binary in -Run.*.CLI requires [!] _Unsafe.SandboxOnly — compiler cannot verify permission compliance"
    - Resolution: Either (1) add `[!] _Unsafe.SandboxOnly` to acknowledge sandbox-only enforcement, or (2) provide source code and switch to -Run.*.Function or -Run.*.Script
    - Example: Show -Run.Go.CLI without _Unsafe.SandboxOnly (error), then with it (passes)
    - Cross-ref: @c:technical/spec/job-sandbox (sandbox spec), @u:permissions/enforcement (user docs)
    - Frontmatter: audience: developer, type: compile-rule

    **PGE10016 — Missing Mandatory Metadata with _Unsafe.SandboxOnly:**
    - Severity: Error
    - Category: Permission / Accountability
    - Trigger: Pipeline using [!] _Unsafe.SandboxOnly but missing one or more of: %Authors, %Description, %Version
    - Message: "_Unsafe.SandboxOnly requires %Authors, %Description, and %Version for accountability"
    - Resolution: Add all three metadata fields to the pipeline definition
    - Example: Show pipeline with _Unsafe.SandboxOnly but missing %Authors (error), then with all three (passes)
    - Cross-ref: @c:technical/spec/job-sandbox
    - Frontmatter: audience: developer, type: compile-rule

    **PGW10007 — _Unsafe.SandboxOnly Active:**
    - Severity: Warning
    - Category: Permission / Compliance
    - Trigger: Pipeline has [!] _Unsafe.SandboxOnly — informational note in compliance report
    - Message: "_Unsafe.SandboxOnly active — maximum sandbox enforcement, suppressed AST-invisible errors listed in compliance report"
    - Content: Lists what additional sandbox layers are activated, notes that PGE10014 errors are suppressed to warnings
    - Cross-ref: @c:technical/spec/job-sandbox, @c:technical/compiler/compliance-report
    - Frontmatter: audience: developer, type: compile-rule

    All three files:
    - Follow H1 heading structure (per #136 convention)
    - Include polyglot code examples
    - Use [[wikilinks]] for Obsidian
