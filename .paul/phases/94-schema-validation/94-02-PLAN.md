<objective>
## Goal
Add `=File.Serial.*` file loading pipelines, `=#.*` schema validation pipelines, and `=#.Field`/`=#.Column` extraction pipelines to the Polyglot pglib specification. Document `<#type` as a pipeline IO pattern extending `<#` from macros to pipelines. Add `!Validation.*`, `!Field.*`, and `!File.ParseError` error leaves.

## Purpose
Closes the runtime validation gap — users need pipelines to load structured data files, validate data against schemas at runtime, extract fields safely, and extract columns from row-oriented Dataframes. This is the core deliverable of Issue #94.

## Output
- `docs/user/pglib/pipelines/File.md` — extended with `=File.Serial.*` pipelines
- `docs/user/pglib/pipelines/#.md` — new file for `=#.*` pipelines (base parsers, validation, field/column extraction)
- `docs/user/pglib/errors/errors.md` — extended with `!Validation.*`, `!Field.*`, `!File.ParseError`
- `docs/user/pglib/INDEX.md` — updated with new pipeline and error entries

- `docs/user/concepts/pipelines.md` — `<#type` cross-reference

</objective>
