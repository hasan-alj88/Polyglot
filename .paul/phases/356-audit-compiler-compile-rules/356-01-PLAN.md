<objective>
## Goal
Normalize rule-file structure across `docs/technical/compile-rules/`:
1. Promote every rule-file heading from H3 (`### Rule X.Y — Name`) to H1, and its `### See Also` trailer to H2, across the 183 files that still use the old template.
2. Correct the stale `rule:` frontmatter values and heading text in 9 of the 10 PGE12xxx files (legacy from the PGE10→PGE12 renumbering).

## Purpose
Issue #356 identifies `h1-title` as the canonical audit convention (`docs/audit/rules/conventions.md` — "H1 = document title"). 34 rule files already match this (from the #136 promotion work); 183 are still on the old H3 template. Closing this gap makes the rule corpus uniform — a precondition for the index files and cross-reference retrofits in plans 356-02 / 356-03.

The PGE12xxx frontmatter is a renumbering artifact from when the range moved from 10.x to 12.x: the file codes (`PGE12001` …) were updated but the internal `rule:` value and heading text still claim `10.1`, `10.2`, etc. This is cosmetic but actively misleading: frontmatter-driven index generation (Plan 356-02) would emit wrong rule numbers.

## Output
- 183 rule files updated with H1 title + H2 "See Also" (PGE + PGW combined).
- 9 PGE12xxx files (PGE12001, 12002, 12003, 12005–12010) with corrected `rule:` frontmatter values and heading text matching the file code.
- No change to body content, frontmatter fields other than `rule:`, or file names.
</objective>
