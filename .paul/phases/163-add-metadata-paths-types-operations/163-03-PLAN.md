## Metadata Path Rules
Pipeline operations use these prefixes based on directory:
- DT/, File/, Math/, Schema/, loose (Path.md, Sys.md): `%definition.=:{name}` / `%=:{name}:N`
- T/: `%definition.T:{name}` / `%T:{name}:N`
- W/: `%definition.W:{name}` / `%W:{name}:N`
- Q/: `%definition.Q:{name}` / `%Q:{name}:N`
- RT/: `%definition.=:{name}` / `%=:{name}:N` (RT pipelines are =RT.* execution pipelines)

The {name} is the full dotted pipeline name (e.g., Math.Add, DT.Parse.ISO, File.Text.Read, T.Call, W.Polyglot, Q.Default).
Extract pipeline name from the H1 heading: `# =Math.Add` → name is `Math.Add`.
For triggers: `# =T.Call` → prefix T, name is `Call`.
For wrappers: `# =W.Polyglot` → prefix W, name is `Polyglot`.
For queues: `# =Q.Default` → prefix Q, name is `Default`.
</context>
