<objective>
## Goal
Split all bundled {#} type definitions into individual per-type files, add metadata tree paths (YAML frontmatter + ## Metadata section) to every type file, and convert the original bundled files into lightweight index pages.

## Purpose
Connects the spec to the `%` metadata tree — every type definition will show its definition path (`%definition.#:{TypeName}`) and instance path (`%#:{TypeName}:N`), making the metadata tree navigable from type docs. Follows from FULL-TREE.md reference.

## Output
- 20 new individual type files (10 enums, 3 structs, 4 collections, 3 runtime)
- 4 bundled files converted to index pages
- 7 existing per-type files updated with metadata paths
- All type files have `metadata_definition` and `metadata_instance` YAML frontmatter + `## Metadata` section
</objective>
