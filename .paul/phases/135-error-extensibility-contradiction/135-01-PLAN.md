<objective>
## Goal
Resolve the contradiction between "only `!Error` is user-extensible" and `{!} !Validation` examples that appear to create new top-level namespaces. Clarify that user `{!} !Name` implicitly nests under `!Error:Name`, update examples, fix the EBNF grammar, and correct the incomplete jm3lib namespace table.

## Purpose
The docs currently mislead readers into thinking `{!}` creates new top-level error namespaces. In reality, user-defined `{!} !Name` creates `!Error:Name.*` in the metadata tree. jm3lib namespaces are separate, fixed, and defined by the runtime. This confusion was flagged by NotebookLM audit (issue #135).

## Output
- Corrected user-facing error docs with accurate `{!}` examples showing implicit `!Error:` nesting
- Updated EBNF grammar that models both jm3lib fixed-leaf and user `!Error:` flexible-branch forms
- Complete jm3lib namespace table (9 namespaces, not 7)
</objective>
