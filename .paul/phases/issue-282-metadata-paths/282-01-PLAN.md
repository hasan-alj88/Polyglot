## Goal
Add `metadata_definition` and `metadata_instance` frontmatter keys plus a Metadata table to all 33 definition files that are currently missing them: 13 error builtin files, 9 scalar type files, and 11 calendar type files.

## Purpose
Issue #282 — definition files should show their full `%` metadata tree path so readers can locate them in FULL-TREE.md. 199 files already follow this pattern; 33 true definition files don't.

## Output
33 modified `.md` files with:
- `metadata_definition` and `metadata_instance` frontmatter keys
- A `## Metadata` section with a table showing definition and instance paths
- Reference to FULL-TREE.md as authoritative source

    For each error file:
    1. Read the file to find the `{!} !Namespace` definition line
    2. Add frontmatter key: `metadata_definition: "%!.Namespace"` (no instance — errors are fixed namespaces)
    3. Add a `## Metadata` section at the end (before `## Related` if present) with this table:

    ```markdown
    ## Metadata

    | Path | Pattern | Description |
    |------|---------|-------------|
    | Tree path | `%!.Namespace` | Fixed error namespace (no instances) |

    See [[metadata-tree/FULL-TREE|Full Metadata Tree]] for the authoritative tree.
    ```

    Error namespace mapping (from FULL-TREE.md):
    - file.md → `%!.File` (.NotFound, .ReadError, .WriteError, .ParseError)
    - math.md → `%!.Math` (.DivideByZero)
    - timeout.md → `%!.Timeout` (.Connection, .Read)
    - validation.md → `%!.Validation` (.Schema, .Type, .Regex)
    - field.md → `%!.Field` (.NotFound, .PathError)
    - no.md → `%!.No` (.Input, .Output)
    - alias.md → `%!.Alias` (.Clash)
    - permission.md → `%!.Permission` (.File.Denied, .Web.Denied, etc.)
    - rt.md → `%!.RT` (.CompileError, .RuntimeError, .Timeout, .EnvironmentError)
    - csv.md, env.md, storage.md, text.md → derive from `{!}` line in file (these are user-extensible under `%!.Error:namespace`)

    Note: csv, env, storage, text may be under `%!.Error:` (user-extensible) not top-level `%!.` — check FULL-TREE.md and each file's `{!}` line to determine correct path.
  </action>
  <verify>

  </verify>
