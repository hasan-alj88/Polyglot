## Goal
Complete the reactive signal table, signal payloads, NATS namespace, and Redis state model to cover all `-Q.*` actions documented in the resource-freeing spectrum and user-facing INDEX.md.

## Purpose

## Design Decisions (confirmed with user)

1. **4 distinct suspended types:** cpu, ram.soft, ram.hard, all — QH knows exact level
2. **One signal + timing field:** command.job.pause.free.cpu {jobId, timing: "now"|"wait"} — QH state write is identical, timing passed to Runner in control signal
3. **Free.All stays in suspended set:** set:suspended with type "all", images_dir on job hash — no new container
4. **All three (throttle/snapshot/inspect) go through QH:** TM is brain, QH initiates all queue commands
5. **Throttle = flag on job hash:** throttled: true/false + throttle_config on job:{jobId}, stays in set:executing
6. **Same resume signal for all types:** QH checks suspended type, if "all" includes images_dir in control signal to Runner
7. **Snapshot: QH auto-enqueues fork:** atomic — generates fork jobId, enqueues in targetQueue, emits control to Runner
8. **Inspect: Redis state only:** returns job:{jobId} hash, resource metrics come from Resource Monitor separately
