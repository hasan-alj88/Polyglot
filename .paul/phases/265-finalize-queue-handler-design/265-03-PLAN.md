    Add a "## Runtime Behavior" section to each action pipeline doc, placed AFTER the existing "## Resource Effects" section (or after "## Errors" if no Resource Effects) and BEFORE "## Permissions".

    Format for each doc (adapted per command):

    ```markdown
    ## Runtime Behavior

    | Step | Component | Action |
    |------|-----------|--------|
    | 1. TM decides | Trigger Monitor | Evaluates rule condition, sends signal |
    | 2. NATS command | `polyglot.command.job.pause.free.cpu.{jobId}` | `{jobId, timing: "now"\|"wait"}` |
    | 3. QH executes | Queue Handler | SREM set:executing, HSET set:suspended "cpu", decrement counters |
    | 4. Control signal | `polyglot.queue.control.{jobId}.job.pause.free.cpu` | `{jobId, timing}` → Runner |
    | 5. Unix mechanism | Runner | `echo 1 > cgroup.freeze` (.Now) or `SIGSTOP` after work unit (.Wait) |
    | 6. Runner ACK | `polyglot.runner.paused.{jobId}` | `{type: "cpu"}` → QH + TM |

    See [[queue-manager/signal-map\|Signal Map]] for the full cross-reference.
    ```

    Adapt the table for each command:
    - Kill commands: SIGTERM/SIGKILL, teardown queue flow
    - Throttle: cgroup cpu.max/memory.high/io.max, no set change
    - Snapshot: criu dump --leave-running, auto-enqueue fork
    - Inspect: Redis read-only, no Unix op
    - Reassign: atomic Redis move (+ CRIU for cross-host)
    - Queue.Drain/Flush: queue-level operations
    - DoNothing: "No runtime signal. Compiler no-op satisfying exhaustiveness."
    - Dispatch.Wait.TimeOut: TM-initiated timeout escalation

    For -Q.Job.Resume, show type-aware routing (cpu/ram → cgroup thaw, all → criu restore).

    Update each file's `updated` frontmatter date.

    Avoid: Changing any existing sections (Definition, Inputs, Outputs, Errors, Resource Effects, Permissions, Metadata). Only ADD the new Runtime Behavior section.
  </action>
