<objective>
## Goal
Unify documentation of `{Q}` dual-purpose behavior: `{Q} #QueueName` (data definition) vs `{Q} =Q.*` (pipeline operation), across user docs, concept docs, and EBNF spec.

## Purpose
`{Q}` is the only block that serves two roles depending on identifier prefix. This is documented in brainstorming (`marker-declarations.md` §0) but not propagated to normative specs. Users reading `queue.md` see zero coverage of the pipeline form. EBNF §9.5 grammar only covers the data form.

## Output
- Updated `blocks.md` — expanded `{Q}` row
- Updated `queue.md` — new "Dual-Purpose `{Q}`" section with examples
- Updated EBNF `09-definition-blocks.md` — clarifying comment/cross-reference in §9.5
</objective>
