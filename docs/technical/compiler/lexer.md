# Polyglot Lexer Architecture

The Polyglot Lexer is responsible for turning a raw `*.pg` script into a completely structured stream of Enum Tokens (`PolyglotToken`). Unlike traditional infinite-character parsers, Polyglot leverages a rigid grammatical structure, allowing the Lexer to operate on a deterministic **Line-by-Line 4-Phase Algorithm**.

This architecture guarantees that the Parser (Compiler Phase 2) always receives extremely predictable token sequences, isolated from arbitrary spacing or line continuation issues.

---

## 1. The 4-Phase Lexing Algorithm

The Engine sweeps directly across each `script.lines()` substring, applying the following phases in exact order:

### Phase 1: Contextual Scope Extraction
Because Polyglot does not use curly brackets `{}` to lock hierarchy, depth is strictly defined by multiples of 3 spaces.
- **Valid Scopes:** The engine counts leading spaces and divides by 3. A single token `Scope(usize)` is pushed to the stream denoting exact hierarchy depth (e.g. `Scope(1)` for 3 spaces).
- **Tab Violations:** If a literal `\t` character is trapped in the leading whitespace, the engine yields `Scope(0)` followed immediately by `IncorrectIndent("\t")`.
- **Space Remainders:** If the leading space string contains an illegal remainder (e.g., 4 or 5 spaces), the engine yields `Scope(1)` followed immediately by `IncorrectIndent(" ")` or `IncorrectIndent("  ")` holding the corrupted substring.

### Phase 2: Marker Trap
Polyglot enforces that every line *must* begin its expression with a bracketed Marker denoting its behavior. The runtime tests for specific `STARTS_WITH` literals:
- `[-]` -> `ActionExecSeq`
- `{#}` -> `DefData`
- `[#]` -> `ActionDataLoad`
- *(If no known Marker is found but the string contains characters, a `MissingMarker` token is yielded to ensure the line retains error-logging fidelity without crashing).*

### Phase 3: The Pattern Registry (Expression Match)
After isolating the Marker, the leading characters are stripped, and the remaining execution string is handed to the **Decoupled Pattern Registry**.

Instead of writing complex abstract syntax algorithms in the main loop, Polyglot evaluates the string against a static, pre-compiled array of Regex macros. 
The registry (`patterns.rs`) iterates sequentially over known patterns (e.g., `^\$(var) +<< +#(data)`) and, upon match, executes a closure to map the Regex captures directly into their underlying `PolyglotToken` outputs (like `Variable`, `OpPushLeft`, `Data`).

*If no pattern matches the string, the engine iterates over each remaining char and swallows them as `TokUnrecognized(c)`, preventing panics and allowing partial AST builds.*

### Phase 4: Line Termination
Every line guarantees that it resolves with a `TokNewline` token, completing the boundary.

## 2. PolyglotToken Registry

The complete catalog of yields generated by the Lexer.

### Structural Yields
- `Scope(usize)`: Unambiguously declares the hierarchy tier for the line.
- `TokNewline`: Explicit end of line marker.
- `TokSpace`: Explicit whitespace required internally inside expression assignments.

### Definitons
Tokenized versions of `Def_` Markers:
- `DefPackage`, `DefData`, `DefPipeline`, `DefTrigger`, `DefWrapper`, `DefNative`, `DefQueue`, `DefError`, `DefPermission`, `DefCollector`, `DefConstructor`, `DefComment`

### Action Elements
Tokenized versions of the Execution/Action `[ ]` Markers:
- `ActionRegistry`, `ActionExecSeq`, `ActionExecPar`, `ActionExecBg`, `ActionDataLoad`, `ActionTypeBind`, `ActionCondSwitch`, `ActionError`, `ActionTrigger`, `ActionQueue`, `ActionWrapper`, `ActionScopeIn`, `ActionScopeOut`, `ActionDataAccessFixed`, `ActionDataAccessFlex`, `ActionLogicalAnd`, `ActionLogicalOr`, `ActionLogicalXor`, `ActionContinuation`, `ActionForeignCode`, `ActionMetadata`, `ActionComment`

### Identifiers (Data-Carrying)
Extracted payloads from `patterns.rs` capturing String values:
- `Variable(String)`: e.g. `$Var`
- `Data(String)`: e.g. `#Data`
- `DataType(String)`: e.g. `#Type`
- `Pipeline(String)`: e.g. `-Pipe`
- `Error(String)`: e.g. `!Err`
- `InputParameter(String)`: e.g. `<param`
- `TerminalData(String)`: e.g. `>terminal`
- `FixedSubField(String)`: e.g. `(>field)`
- `FlexibleSubField(String)`: e.g. `(<field)`

### Operators
Standard Polyglot directional operators:
- `OpPushLeft`: `<<`
- `OpPushRight`: `>>`
- `OpFallbackPushLeft`: `<<?`
- `OpFallbackPushRight`: `>>?`

### Error Isolation & Safety
Tokens used to swallow malformed scripts without panicking:
- `MissingMarker`: Tripped when Phase 2 captures no bracket syntax.
- `IncorrectIndent(String)`: Tripped when Phase 1 traps trailing spaces or tabs.
- `TokUnrecognized(char)`: Driven when Phase 3 cannot extract a known Macro, isolating the specific character.
