# Test Design: Story 1.5 - Recursive Descent Parser Implementation

**Date:** 2025-11-28
**Designer:** Quinn (Test Architect) via Team Discussion
**Story:** 1.5 - Recursive Descent Parser Implementation
**Status:** Draft

---

## Test Strategy Overview

- **Total test scenarios:** 94 (planned)
- **Unit tests:** 60 (64%)
- **Integration tests:** 28 (30%)
- **E2E tests:** 6 (6%)
- **Priority distribution:** P0: 70, P1: 18, P2: 6, P3: 0

### Scope Clarification

**Story 1.5 covers:**
- ✅ Single-file parsing (`.pg` file → AST)
- ✅ Phase 1 resolution: Current file namespace
- ✅ Phase 3 resolution: External packages via FileRegistryResolver
- ✅ Error detection for all 37 ParserError types
- ✅ Parameter validation (required/optional, defaults)
- ✅ Type checking for pipeline inputs/outputs

**Out of Scope (Deferred to Story 1.5.5):**
- ❌ Multi-file same-package resolution (Phase 2)
- ❌ File ordering validation `[#]`
- ❌ Combined symbol tables across files

### Test Levels Rationale

Per test-levels-framework:
- **Unit tests:** Individual parser functions (parse_pipeline, parse_block, parse_expression, etc.)
- **Integration tests:** Lexer → Parser flow with real Polyglot code
- **E2E tests:** Complete .pg file parsing with registry resolution

## Implementation Notes

### Parser Structure (Recommended)

```rust
pub struct Parser {
    tokens: Vec<Token>,
    current: usize,
    resolver: Box<dyn ImportResolver>,
}

impl Parser {
    pub fn parse_file(path: &str, resolver: Box<dyn ImportResolver>) -> Result<Program> {
        let source = fs::read_to_string(path)?;
        let tokens = Lexer::new(&source).tokenize()?;
        let mut parser = Parser::new(tokens, resolver);
        parser.parse_program()
    }

    fn parse_program(&mut self) -> Result<Program> {
        let package = self.parse_package_declaration()?;
        let definitions = self.parse_definitions()?;
        Ok(Program { package, definitions, .. })
    }

    fn parse_package_declaration(&mut self) -> Result<PackageDeclaration> { .. }
    fn parse_definitions(&mut self) -> Result<Vec<Definition>> { .. }
    fn parse_pipeline(&mut self) -> Result<Pipeline> { .. }
    fn parse_enumeration(&mut self) -> Result<EnumerationDefinition> { .. }
    fn parse_error_definition(&mut self) -> Result<ErrorDefinition> { .. }
    // ... more parse functions
}
```
