## Critical Rules from Authoritative Reference

These rules from [quick-language-reference.md](quick-language-reference.md) are AUTHORITATIVE:

1. **Error Raising:** MUST use `[r] .error: ! << !ErrorType` then field assignments then `[o] .error: !`
2. **Generic Error Type:** `!` (exclamation mark alone), specific types like `!ValidationError`
3. **NO Comparison Operators:** `==`, `!=`, `<`, `>`, `<=`, `>=` do NOT exist - use range notation
4. **Range Notation:** `.value..` (greater than), `...value` (less than), `.value` (equals)
5. **String Continuation:** Requires `[^]` block marker + `+"` prefix on continuation lines
6. **Trigger Requirement:** ALL pipelines MUST have `[t]` triggers
7. **Type Separator:** Always backslash `\` (never forward slash `/`)
8. **Assignment Operators:** `<<` (push INTO), `>>` (pull FROM)
9. **Path Identifiers:** `#Path.Identifiers.*` for cross-platform paths
10. **No Explicit !NoError:** `!NoError` exists implicitly but is NEVER explicitly checked or assigned

#### Violation E1: Incorrect Array Type Syntax (7 instances)

**Lines:** 326, 331, 332, 379, 382, 385, 387

**Problem:** Uses square brackets `[` `]` for array type syntax instead of curly braces `{` `}`

**Incorrect:**

**Correct:**
```polyglot

[r] .results: pg\array{pg\string} << []
[r] .errors: pg\array{!} << []
```

**Authoritative Reference:** Lines 311-318 in quick-language-reference.md

## Recommendations

### Priority 1: CRITICAL - Fix Comparison Operators (12 violations)

Replace ALL instances of comparison operators with:
- Range notation: `.value..` (greater), `...value` (less), `.value` (equal)
- Match operator: `?>`
- Switch patterns: `[?]` blocks

**Impact:** These violations represent code that would NOT compile

### Priority 2: CRITICAL - Remove Explicit !NoError Checks (2 violations)

Remove all explicit `!NoError` comparisons and checks. The language philosophy is:
- Default behavior is success
- Only catch SPECIFIC error types
- No need to check for "no error"

**Impact:** Violates language design philosophy

### Priority 3: HIGH - Fix Array Type Syntax (7 violations)

Replace `pg\array[type]` with `pg\array{type}` throughout approved examples

**Impact:** Syntax errors that would not compile

### Priority 4: HIGH - Fix Parallel/Join Syntax (2 violations)

Fix parallel block and join marker syntax in approved examples

**Impact:** Examples teach incorrect patterns

### Priority 5: MEDIUM - Fix Multiline String Syntax (1 violation)

Correct continuation prefix from `+""` to `+"`

**Impact:** Minor syntax error in examples
