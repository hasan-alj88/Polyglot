## Implementation Guidance

### Priority 1: Lexer/Parser Implementation

**Use exclusively:**
- [v0.0.2 BNF Grammar](../language/bnf/polyglot grammer.md)
- Examples: [example.pg](../../../Polyglot/Lexer/example.pg) (now v0.0.2 compliant)

**Key lexer decisions:**
- Type separator: `\` (backslash)
- Comment start: `//` (forward slash)
- Assignment: `<<` and `>>` (directional arrows)
- Variable prefix: `.` (dot, mandatory)
- Block markers: Case-sensitive `[X]`, `[Q]`, etc.

### Priority 2: Type System

**Follow v0.0.2 decisions:**
- Basic types: `pg\int`, `pg\string`, `pg\bool`, `pg\dt`, `pg\path`
- Mutable types: `pg.mutable\int`
- Collections: `pg\array{}`, `pg\set{}`
- **NO MAPS** - Use enumerations instead (Decision #1)
- DateTime system: `DT"..."` prefix only

### Priority 3: Database Schema

**Use v0.0.1 schema design:**
- [Database Schema](../../v0.0.1/planning/architecture/database-schema.md)
- IR stored as relational tables (not JSON blobs)
- AST tree structure in `ir_nodes` table
- Type information stored as strings: `'py\int'`, `'pg\dt'`
