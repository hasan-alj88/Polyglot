# =# тАФ Schema Validation & Field Extraction

The `=#` namespace groups schema-related pipelines. The `=` is the pipeline prefix; `#` is the first name segment (referring to schema/type operations). This follows the same convention as `=W.*` (wrappers), `=Q.*` (queues), and `=T.*` (triggers) тАФ no prefix collision.

Stdlib pipelines for schema validation, field extraction, format parsing, and dataframe column extraction. No `[@]` import needed. See [[errors#Pipeline Error Associations]] for error contracts.

Types are data trees (GT-1). The `<#type` syntax passes a type definition's `%` metadata tree as pipeline input тАФ extending `<#` from `{M}` macro type inputs to `{=}` pipeline IO. Works with any tier: `#Config` (struct), `##Scalar` (schema), `###Enum` (property). See [[syntax/types/macro-types#<#type in Pipeline IO]].

```polyglot
=#
   .JSON.Parse
      <raw#RawString
      >data#serial
   .YAML.Parse
      <raw#RawString
      >data#serial
   .TOML.Parse
      <raw#RawString
      >data#serial
   .Match
      <data#serial
      <#type
      >match#bool
   .Validate
      <data#serial
      <#type
      >valid#bool
      >errors#array:string
   .Describe
      <#type
      >schema#serial
   .Coerce
      <data#serial
      <#type
      >result#serial
      >dropped#array:string
   .Field
      <data#serial
      <path#RawString
      >value#serial
   .Column
      <data#dataframe
      <column#enum
      >values#array
```

## Native Parsers (Compiler Intrinsics)

`=#.JSON.Parse`, `=#.YAML.Parse`, and `=#.TOML.Parse` are compiler-internal implementations that convert raw text to `#serial` data trees. They are not user-definable тАФ the compiler provides them as intrinsics.

### `=#.JSON.Parse`

Parses a JSON string into a `#serial` data tree.

| IO | Type | Description |
|----|------|-------------|
| `<raw` | `#RawString` | Raw JSON text |
| `>data` | `#serial` | Parsed data tree |

No errors тАФ invalid input handled by calling pipeline (e.g. `=File.Serial.Read` raises `!File.ParseError`).

### `=#.YAML.Parse`

Parses a YAML string into a `#serial` data tree.

| IO | Type | Description |
|----|------|-------------|
| `<raw` | `#RawString` | Raw YAML text |
| `>data` | `#serial` | Parsed data tree |

No errors тАФ same as `=#.JSON.Parse`.

### `=#.TOML.Parse`

Parses a TOML string into a `#serial` data tree.

| IO | Type | Description |
|----|------|-------------|
| `<raw` | `#RawString` | Raw TOML text |
| `>data` | `#serial` | Parsed data tree |

No errors тАФ same as `=#.JSON.Parse`.

## Schema Validation Pipelines

These pipelines use `<#type` тАФ a type definition passed as a data tree input. The pipeline reads the type's `%` metadata (schema properties, field definitions, constraints) to validate or introspect data at runtime.

### `=#.Match`

Boolean schema check тАФ does this data match this type? Returns `#True`/`#False`, no side effects.

| IO | Type | Description |
|----|------|-------------|
| `<data` | `#serial` | Data to check |
| `<#type` | type tree | Type definition as data tree input |
| `>match` | `#bool` | Whether data matches the type's schema |

No errors тАФ always succeeds.

### `=#.Validate`

Detailed validation тАФ checks data against type and reports all mismatches as a list of human-readable error strings.

| IO | Type | Description |
|----|------|-------------|
| `<data` | `#serial` | Data to validate |
| `<#type` | type tree | Type definition as data tree input |
| `>valid` | `#bool` | Overall pass/fail |
| `>errors` | `#array:string` | List of validation error descriptions |

No errors тАФ always succeeds. Validation failures are reported via `>errors`, not `[!]`.

### `=#.Describe`

Schema introspection тАФ returns a type's full schema (properties, fields, constraints) as a `#serial` data tree. Useful for runtime reflection.

| IO | Type | Description |
|----|------|-------------|
| `<#type` | type tree | Type definition to describe |
| `>schema` | `#serial` | Full schema as data tree |

No errors тАФ always succeeds.

### `=#.Coerce`

Best-effort type conversion тАФ takes data and a target type, keeps fields that match, reports fields that don't. Always succeeds (never throws).

| IO | Type | Description |
|----|------|-------------|
| `<data` | `#serial` | Data to coerce |
| `<#type` | type tree | Target type |
| `>result` | `#serial` | Coerced data (matching fields only) |
| `>dropped` | `#array:string` | List of field paths that couldn't be coerced |

No errors тАФ always succeeds.

## Field & Column Extraction

### `=#.Field`

Extracts a single field from a `#serial` data tree by path. Like tree access (`$data<key<subkey`) but with error handling for missing paths.

| IO | Type | Description |
|----|------|-------------|
| `<data` | `#serial` | Data tree to extract from |
| `<path` | `#RawString` | Tree path using `<` separator (e.g. `"database<host"`) |
| `>value` | `#serial` | Extracted value |

| Error | When |
|-------|------|
| `!Field.NotFound` | Path doesn't exist in data tree |
| `!Field.PathError` | Invalid path syntax |

**Three approaches to field access тАФ user picks based on needs:**

| Approach | When to use |
|----------|-------------|
| `$data<database<host` | Direct tree access тАФ fast, no error handling on missing field |
| `=#.Field` | Safe extraction тАФ error handling with `[!]` fallback chains |
| `=File.Serial.Read.Field` | Single step from file to field тАФ combines read + parse + extract |

### `=#.Column`

Extracts all values for a given column across all rows of a row-oriented Dataframe, returning them as an array. Solves the column access pattern that row-oriented storage doesn't natively provide.

| IO | Type | Description |
|----|------|-------------|
| `<data` | `#dataframe` | Source dataframe |
| `<column` | `#enum` | Column identifier from the dataframe's column enum |
| `>values` | `#array` | All values in that column, one per row |

| Error | When |
|-------|------|
| `!Field.NotFound` | Column doesn't exist in dataframe's column enum |

See also: [[concepts/collections/expand#Expand Operators]], [[syntax/types/macro-types#Macro-Generated Types]]

## Permissions

Validation and extraction pipelines perform pure computation тАФ no `{_}` permission objects needed. Native parsers are compiler intrinsics and have no permission requirements.

## Errors

```polyglot
=#.Field
   !Field.NotFound
   !Field.PathError

=#.Column
   !Field.NotFound
```

All other `=#.*` pipelines are non-failable тАФ no error declarations.
