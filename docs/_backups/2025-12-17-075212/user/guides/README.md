## Guide Format

All guides in this directory follow a consistent format:

1. **Introduction** - What this guide covers
2. **Prerequisites** - What you need to know first
3. **Step-by-Step Instructions** - Detailed walkthrough
4. **Examples** - Before/after code samples
5. **Common Issues** - Troubleshooting
6. **Next Steps** - Where to go from here

## Contributing Guides

When writing new guides:

1. **Focus on practical tasks** - Help users accomplish specific goals
2. **Provide complete examples** - Show full, working code
3. **Explain the why** - Don't just show what to do
4. **Include troubleshooting** - Address common problems
5. **Link to related docs** - Connect to broader documentation
