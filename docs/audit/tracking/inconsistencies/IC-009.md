# IC-009: Vision vs. Syntax (Logic Creep)

**Severity:** High
**Tag:** vision
**File(s):** [[vision.md]], [[user/syntax/constructors.md]]
**Resolved Date:** 2026-04-23

## Description
There is a fundamental contradiction between the Project Vision and the Constructor implementation. 
- **Vision:** Polyglot is the "Glue, not the Bricks." It should not reinvent what legacy languages do.
- **Syntax:** `constructors.md` implements complex regex parsing and mapping, effectively reinventing string parsers inside the "Glue" layer.

## Resolution
Restricting `{$}` constructors to literal mapping or very simple patterns, and pushing complex parsing (like Semver) to external language "Bricks". `constructors.md` was rewritten to remove `($)` explicit `.re` regex parameter definition lines. Capture slots now inherit structural constraints completely from their mapped fields while still retaining full compile-time checks and generating no error surfaces.
