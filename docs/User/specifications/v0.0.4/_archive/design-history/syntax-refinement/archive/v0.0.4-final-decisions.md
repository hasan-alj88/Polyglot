# Polyglot v0.0.4 - Final Design Decisions (Updated)

**Date:** 2025-12-11
**Status:** ‚úÖ UPDATED - Key Decisions Made

---

## Critical Design Decisions

### 1. Reserved Indication System

**Decision:** Use `;` (semicolon) for reserved schema parts on `#` and `!`

**Syntax:**
```polyglot
// Fully reserved
#;Boolean;True
#;Boolean;False

// Mixed reserved/user
#;DT;Business.MyCompanyWeek;WorkingDays
//   ^^       ^^^^^^^^^^^^^ ^^^^^^^^^^^
// reserved  user-defined   reserved

// Fully user-defined (no semicolons)
#OrderStatus.Processing
```

**Rules:**
- `;` marks reserved segment
- `.` marks user-defined segment
- Can mix in same path for extensions
- Use `%Enumeration.reserved` metadata (core Polyglot devs only)

**Applies to:**
- `#` Enums: `#;Boolean;True`
- `!` Errors: `!;Network;HTTP;Timeout`

**Does NOT apply to:**
- `:` Types: Still use `:pg.int` (subject to change later)
- Other prefixes: Use `.` for all hierarchy

**Benefits:**
- ‚úÖ Granular segment-level control
- ‚úÖ Users can extend reserved namespaces
- ‚úÖ Clear visual distinction
- ‚úÖ IDE can provide intelligent autocomplete

---

### 2. Mutability: Strictly NO

**Decision:** Polyglot does NOT deal with mutable variables at all

**Rejected:**
- ‚ùå `[v]` mutable variable marker
- ‚ùå Pack/unpack operators
- ‚ùå Reassignment operators
- ‚ùå Update operators
- ‚ùå Shadowing

**Solution: Functional composition**
```polyglot
// Sequential variables
[r] $step0 << "initial"
[r] $step1 << |Transform1 <input << $step0
[r] $step2 << |Transform2 <input << $step1

// Or pipeline composition
[r] |Transform1
   <input << "initial"
   |> |Transform2
   |> |Transform3
   >output >> $result

// Or chained loops (see below)
```

**Rationale:**
- Immutability is core to Polyglot's design
- Functional programming patterns solve transformation problems
- No state management issues
- Safer and more predictable

---

### 3. Pipeline as Variable: On Hold

**Decision:** Putting `:pg.pipeline` on hold

**Reason:** Calling pipelines from variables is a slippery slope for mismanagement

**Status:** Find other workarounds for now

**Impact:**
- Cannot store pipeline references in variables (for now)
- Cannot pass pipelines as parameters (for now)
- Cannot return pipelines from pipelines (for now)
- Will revisit after core features stabilize

### 4. Loop I/O & Mini-Pipelines: NEW SYSTEM

**Decision:** Each loop iteration is a mini-pipeline with explicit I/O

### Loop I/O Operators

**New operators: `l<` and `l>`**

```polyglot
[<] l<array << $array_var              // Loop input
[>] l>item >> $item_var                // Loop output (iteration variable)
[>] l>out_loop << ...                  // Iteration output
```

### Execution Modes

| Marker | Mode | Description |
|--------|------|-------------|
| `[r] ~*` | Sequential | Iterations run one after another |
| `[p] ~*` | Parallel | Iterations run concurrently (must join) |
| `[b] ~*` | Fire-and-forget | No output expected |

### Sequential Example

```polyglot
[r] ~ForEach
   [<] l<array << $items
   [>] l>item >> $item

   [r] $processed << |Transform <input << $item
   [>] l>out_loop << $processed

[>] o>results << $out_loop:array.serial
```

### Parallel Example (Requires Join)

```polyglot
[p] ~ForEach
   [<] l<array << $items
   [>] l>item >> $item

   [r] $processed << |ExpensiveOp <input << $item
   [>] l>out_loop << $processed

   [V] ~V.JoinAll                      // REQUIRED: join point
      [<] <out_loop
      [>] >results >> $all_results

[>] o>results << $all_results:array.serial
```

### Chained Loop (Sequential Only)

```polyglot
[r] ~ForEach.Chained
   [<] l<array << $steps
   [<] l<seed << $initial_value
   [>] l>item >> $step

   [r] $step
      <input << $seed
      >output >> $result

   [>] l>out_loop >> l<seed            // ‚≠ê Chain to next iteration

[>] o>final << $seed:serial
```

**Key:** `[>] l>out_loop >> l<seed` - output becomes input for next iteration

**Compile error:**
```polyglot
[p] ~ForEach.Chained                   // ‚ùå ERROR: Cannot chain parallel
```

### Loop Patterns

**Standard patterns:**
- `~ForEach` - Standard iteration
- `~Enumerate` - With index
- `~ForEach.Chained` - Chain output to next (sequential only)
- `~Range` - Numeric iteration
- `~While` / `~Until` - Conditional loops
- `~Reduce` - Accumulator/fold
- `~Recursive.Tournament.*` - Tree reduction

**Functional patterns:**
- `~Map` - Transform each item
- `~Filter` - Keep matching items
- `~FlatMap` - Flatten nested arrays
- `~TakeWhile` / `~DropWhile` - Conditional take/drop
- `~Partition` - Split by predicate
- `~Zip` - Combine two arrays
- `~GroupBy` - Group by key

### Join/Sync Patterns

**Required for `[p]` parallel loops:**

| Pattern | Description |
|---------|-------------|
| `~V.JoinAll` | Wait for all iterations to complete |
| `~V.JoinFirst` | Race - return first to complete |
| `~V.JoinAny` | Return first success (ignore failures) |
| `~V.RecurseUntilOne` | Tree reduction until single value |

### Reduction Example

```polyglot
[r] ~Reduce
   [<] l<array << $numbers
   [<] l<initial << 0
   [>] l>item >> $num
   [>] l>accumulator >> $acc

   [r] $new_acc << $acc + $num

   [>] l>accumulator << $new_acc       // New accumulator (not mutation!)

[>] o>sum << $accumulator:int
```

### Tournament Reduction (Parallel)

```polyglot
[p] ~Recursive.Tournament.RandomPair
   [<] l<array << $values
   [>] l>pair >> $pair

   [r] $sum << $pair.0 + $pair.1
   [>] l>reduced << $sum

   [V] ~V.RecurseUntilOne
      [<] <reduced
      [>] >final >> $total

[>] o>total << $total:int
```

**How it works:**
- Pair items: `[1,2,3,4,5,6,7,8]` ‚Üí `(1,2), (3,4), (5,6), (7,8)`
- Process in parallel: `[3, 7, 11, 15]`
- Recurse: `(3,7), (11,15)` ‚Üí `[10, 26]`
- Recurse: `(10,26)` ‚Üí `[36]`
- Done!
