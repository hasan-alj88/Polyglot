# Polyglot v0.0.4 - Complete Syntax Specification

**Date:** 2025-12-11
**Status:** ✅ FINAL - Ready for Implementation

---

## Core Principles

1. **No keywords** - Only markers and operators
2. **One line = One marker + One expression**
3. **Indentation for nesting** (3 spaces)
4. **Universal hierarchy** - `PREFIX.identifier.hierarchy`
5. **Explicit over implicit**
6. **Variable prefix: `$`** (not `,` due to separator ambiguity)

---

## Prefix Operators

| Prefix | Purpose | Example |
|--------|---------|---------|
| `$` | Variable/parameter | `$user`, `$user.profile.name` |
| `:` | Type path | `:pg.string`, `:array.int` |
| `#` | Enum/struct | `#OrderStatus.Processing` |
| `\|` | Pipeline | `\|Database.Users.Find` |
| `!` | Error | `!Network.HTTP.Timeout` |
| `@` | Registry | `@Local::MyApp:1.0.0.0` |
| `%` | Metadata | `%Doc`, `%Author` |

## I/O Operators - Critical Distinction

### Pipeline DEFINITION I/O: `i<` and `o>`

```polyglot
{|} |Pipeline.Example
[<] i<input_param:string               // Pipeline input parameter
[>] o>output_param:string              // Pipeline output parameter
{x}
```

### Pipeline CALL I/O: `<` and `>`

```polyglot
[r] |Pipeline.Example
   <input_param << $my_value           // Pass input
   >output_param >> $result            // Capture output
```

### Markers Always Mean I/O

- `[<]` - Input marker (always for I/O)
- `[>]` - Output marker (always for I/O)

**This distinction is critical!**
- `i<` / `o>` = defining pipeline signature
- `<` / `>` = calling pipeline

## Multi-line Strings: `[+]` Marker

**Syntax:**
```polyglot
[r] $query << |SQL""                   // Start inline pipeline
[+] +"SELECT * FROM orders"            // Each line marked with [+] +"..."
[+] +"WHERE order_id = "
[+] +|SQL"{$order_id}"                 // Can interpolate inline pipelines
[+] +"  AND status = 'pending'"
```

**Benefits:**
1. Each line explicitly marked (safeguard)
2. Prevents Python's silent concatenation bug
3. Can mix literals and inline pipelines
4. Parallel evaluation, then safe concatenation

**Inline Pipeline Types (for future implementation):**
- `|SQL"{...}"` - SQL injection prevention
- `|HTML"{...}"` - HTML entity escaping
- `|JSON"{...}"` - JSON-safe escaping
- `|URL"{...}"` - URL encoding
- `|Regex"{...}"` - Regex escaping
- `|Shell"{...}"` - Shell command escaping
- `"{...}"` - Direct interpolation (no escaping)

## Inline Pipelines (Not "String Literals")

**Definition:**
Inline pipelines are single-output pipelines that evaluate in place.

**Structure:**
```polyglot
{|} |Inline.Pipeline.Name
[<] i<formatted_string:string          // Input parameter

[t] |T.Call.Inline                     // Special trigger for inline pipelines

[W] |W.Implementation                  // Implementation

[>] o>output:string                    // MUST have exactly one output
{x}
```

**Usage:**
```polyglot
[r] $result << |Double"{$value}"       // Inline evaluation
[r] $timestamp << |DT.Now""            // No parameters
[r] $safe_sql << |SQL"{$user_input}"   // SQL-safe escaping
```

**Rules:**
- ✅ Must have exactly one output
- ✅ Evaluates inline and replaces itself with output
- ✅ Can be used in expressions

## Inline Error Pipelines

**Syntax sugar for error throwing:**
```polyglot
[>] o>error << !InvalidOrder"Invalid priority level"
```

**Expands to:**
```polyglot
[>] o>error << !InvalidOrder
   <message << "Invalid priority level"
```

**General form:**
```polyglot
!ErrorType"message"                    // Short form
!ErrorType <param << value             // Full form
```

## Collection Literals

### Arrays
```polyglot
[r] $tags:array.string << {"web", "api", "v2"}
[r] $numbers:array.int << {1, 2, 3, 4, 5}
```

### Sets
```polyglot
[r] $unique_numbers:set.int << {1, 2, 3, 4, 5}
```

**Type determines if array or set based on context.**

## Range Operators

### Inclusive/Exclusive Ranges
```polyglot
[f] $age ?[18, 65]                     // Inclusive: 18 <= age <= 65
[f] $age ?(18, 65]                     // Exclusive min: 18 < age <= 65
[f] $age ?[18, 65)                     // Exclusive max: 18 <= age < 65
[f] $age ?(18, 65)                     // Exclusive both: 18 < age < 65
```

### Range in Collections
```polyglot
[f] $priority in? $valid_priorities    // Check if in collection
[f] $priority !in? $invalid_list       // Check if NOT in collection
```

### Iteration Ranges
```polyglot
[p] ~ForEach.Range"1..100"             // Iterate 1 to 100
   >index >> $i
   [r] $squared << $i * $i
```

## Operator Negation: `!?`

**All operators with `?` can be negated with `!?`:**

```polyglot
// Equality
[f] $status =? "active"                // Equals
[f] $status !=? "active"               // Not equals

// Comparison
[f] $age >? 18                         // Greater than
[f] $age !>? 65                        // NOT greater than (<=)

// Range
[f] $age ?[18, 65]                     // In range
[f] $age !?[18, 65]                    // NOT in range

// Collection membership
[f] $role in? $valid_roles             // In collection
[f] $role !in? $invalid_roles          // NOT in collection

// Pattern matching
[f] $email re? ".*@example.com"        // Regex match
[f] $email !re? ".*@spam.com"          // Regex NOT match
```

## Variadic Input: `<<<` Shorthand

**Enable via metadata:**
```polyglot
{|} |Set.Intersection
[<] i<type:pg.type                     // Element type
[<] i<sets:serial                      // Dynamic array (type unknown at compile)
   [%] %InStream << #IO.Stream.TriplePull    // Enable <<< operator
   [%] %Keys << #IO.Stream.Keys.numeric      // Numeric keys (0, 1, 2, ...)
   [%] %ItemType << $type                    // Item type for validation

[t] |T.Call
[W] |W.Polyglot.Scope

[p] ~Recursive.Tournament.RandomPairs  // Custom recursive algorithm
   <array << $sets
   >item1 >> $set1
   >item2 >> $set2
   [r] $result << |Set.IntersectTwo
      <set1 << $set1
      <set2 << $set2
      >intersection >> $result
   [>] >winner << $result

[>] o>intersection << $result:set
{x}
```

**Usage:**
```polyglot
[r] |Set.Intersection
   <<< $set1                           // Appends to sets array
   <<< $set2
   <<< $set3
   >>> $intersection                   // Captures output
```

**Metadata controls:**
- `%InStream` - Enables the shorthand operator (`#IO.Stream.TriplePull` = `<<<`)
- `%Keys` - How items are indexed (numeric, named, etc.)
- `%ItemType` - Type validation for streamed items

## Early Return

**Using output marker with `o>` operator:**
```polyglot
[f] $input =? ""
   [>] o>error << !InvalidInput"Empty input"

[f] $priority !in? $valid_priorities
   [>] o>error << !InvalidOrder"Invalid priority level"
```

**Semantics:**
- `[>] o>variable` = output and return immediately
- Only valid inside conditionals
- Must match a defined output parameter

## Metadata System: `%` Prefix

### Registry Metadata
```polyglot
{@} @Local::MyApp:1.0.0.0
[A] @App
[<] @PgTypes                           // Import alias registry

[%] %Doc <<
[+] +"Advanced application for"
[+] +"managing user workflows"

[%] %Author
   [.] .name << "Alice <alice@example.com>"
   [.] .since << "0.1.0"

[%] %Author                            // Multiple authors allowed
   [.] .name << "Bob <bob@example.com>"
   [.] .since << "0.1.4"

[%] %License << "MIT"
[%] %Repository << "https://github.com/myapp/repo"
{x}
```

### Pipeline Metadata
```polyglot
{|} |FetchUser
[%] %PublicTo << {@MyApp}              // Export visibility
[%] %PrivacyGroup << "Authentication"

[%] %Errors
   [.] .error << @MyApp!Database.Timeout
   [.] .when << "Database unreachable"

[<] i<user_id:string
   [%] %description << "Unique user identifier"
   // %Params.0.name and %Params.0.type inferred from declaration

[<] i<include_profile:bool
   [%] %description << "Include full profile data"
   [%] %default << #Boolean.False

[>] o>user:serial
   [%] %description << "User data with optional profile"

{x}
```

### Deprecation Metadata
```polyglot
{|} |FetchUser
[%] %Deprecated
   [.] .new_pipeline << |FetchUserV2
   [.] .message << "Use |FetchUserV2 instead"
   [.] .auto_migrate << #Boolean.True  // Auto-switch in tooling
{x}
```

### Standard Metadata Attributes
- `%Doc` - Documentation (multi-line with `[+]`)
- `%Author` - Author information (repeatable)
- `%Deprecated` - Deprecation notice with migration path
- `%PublicTo` - Export visibility control
- `%PrivacyGroup` - Privacy/security grouping
- `%Errors` - Expected errors with conditions
- `%License` - License information
- `%Repository` - Source repository
- `%Since` - Version introduced
- `%Example` - Usage examples
- `%InStream` - Variadic input configuration
- `%ItemType` - Type validation for streams

**Metadata references:**
```polyglot
{@MyApp%PrivacyGroup.Authentication}   // Reference metadata
@OrderProc!InvalidOrder                // Reference error from registry
```

## Enum Definitions (File Scope)

**Enums are defined at file scope:**
```polyglot
{@} @Local::Types:1.0.0.0
[A] @Types
{x}

// File-scope enum with value fields
{#} #OrderResult
[.] .success
   [.] .order_id:string                // Value field
   [.] .tracking:string                // Value field
[.] .error
   [.] .code:int
   [.] .message:string
{x}

// File-scope simple enum
{#} #Priority
[.] .high
[.] .medium
[.] .low
{x}

// Pipeline can use enums defined above
{|} |ProcessOrder
...
{x}
```

**Rule:** Enum field siblings must be enums, but can contain value subfields.

## Struct Field Shorthand

**When variable name matches field name:**
```polyglot
{#} #Person
[.] .name:string
[.] .email:string
[.] .age:int
[.] .created:dt
{x}

[r] $name << "Alice"
[r] $email << "alice@example.com"
[r] $age << 30
[r] $timestamp << |DT.Now""

[r] $person << #Person
   [.] .name                           // Shorthand: << $name
   [.] .email                          // Shorthand: << $email
   [.] .age                            // Shorthand: << $age
   [.] .created << $timestamp          // Explicit: different variable name
```

**Rule:** If no `<<` operator after field, assumes `<< $<fieldname>`

---

## Struct Auto-filling from Pipeline

**Pipeline can output struct and auto-fill matching fields:**
```polyglot
[r] @OrderData|GetOrder
   <order_id:string << $order_id
   >record:array.serial >> $order_record:#OrderRecord
   // Auto-fills fields with matching names from output

// Then modify specific fields
[r] $order_record << $order_record
   [.] .priority << $priority_level    // Override/set specific field
   [.] .updated << |DT.Now""           // Add new field
```

---

## Match Expressions (Exhaustive)

**All match expressions must be exhaustive:**
```polyglot
[m] $processing_type << $priority      // Variable needs $ prefix!
   [?] "high" ? "express"
   [?] "medium" ? "standard"
   [?] "low" ? "economy"
   [?] * ? "standard"                  // Wildcard required for exhaustiveness
```

**Compiler enforces:**
- All cases covered
- Wildcard `*` for catch-all
- No unreachable cases

## Complete Example

```polyglot
{@} @Local::OrderProcessing:1.0.0.0
[A] @OrderProc
[<] @PgTypes
[<] @OrderData << @Local::OrderDBOps:1.0.0.0

[%] %Doc <<
[+] +"Order processing module with"
[+] +"validation and error handling"

[%] %Author
   [.] .name << "Alice <alice@example.com>"
   [.] .since << "1.0.0"
{x}

// ============================================================================
// File-scope enum definitions
// ============================================================================

{#} #OrderResult
[.] .success
   [.] .order_id:string
   [.] .tracking:string
[.] .error
   [.] .code:int
   [.] .message:string
{x}

{#} #Priority
[.] .high
[.] .medium
[.] .low
{x}

// ============================================================================
// Process Order Pipeline
// ============================================================================

{|} |ProcessOrder
[%] %PublicTo << {@OrderProc}
[%] %Errors
   [.] .error << @OrderProc!InvalidOrder
   [.] .when << "Order validation fails"

[<] i<order_id:string
   [%] %description << "Unique order identifier"
   // %Params.0.name and %Params.0.type inferred

[<] i<priority:string
   [%] %description << "Order priority: high, medium, or low"

[t] |T.Call
[W] |W.Polyglot.Scope

// Multi-line SQL with injection safety
[r] $query << |SQL""
[+] +"SELECT * FROM orders"
[+] +"WHERE order_id = "
[+] +|SQL"{$order_id}"
[+] +"  AND status = 'pending'"

// Collection literals
[r] $valid_priorities << {"high", "medium", "low"}
[r] $admin_tags << {"admin", "priority", "urgent"}

// Range validation with in? operator
[f] $priority in? $valid_priorities
   [r] $valid_priority << #Boolean.True
[f] *?
   [>] o>error << !InvalidOrder"Invalid priority level"  // Early return

// Various range validations
[f] $customer.age ?[18, 65]            // Inclusive both
   [r] $age_valid << #Boolean.True
[f] $customer.age ?(18, 65)            // Exclusive both
   [r] $young_adult << #Boolean.True

// Set operations with <<< shorthand
[r] $user_tags << {"user", "premium", "admin"}
[r] |Set.Intersection
   <<< $admin_tags
   <<< $user_tags
   >>> $common_tags

// Match expression (exhaustive)
[m] $processing_type << $priority      // $ prefix required!
   [?] "high" ? "express"
   [?] "medium" ? "standard"
   [?] "low" ? "economy"
   [?] * ? "standard"                  // Exhaustive catch-all

// Struct auto-filling from pipeline
[r] @OrderData|GetOrder
   <order_id:string << $order_id
   >record:array.serial >> $order_record:#OrderRecord

// Struct field modifications with shorthand
[r] $order_id << "ORD-12345"
[r] $customer_email << "alice@example.com"
[r] $priority_level << $processing_type

[r] $order_record << $order_record
   [.] .order_id                       // Shorthand: << $order_id
   [.] .priority << $priority_level    // Explicit: different name
   [.] .email << $customer_email       // Explicit
   [.] .created << |DT.Now""           // Inline pipeline

// Inline pipeline transformation
[r] $doubled_priority << |Double"{$priority_level}"

[>] o>order_record << $order_record:#OrderRecord
[>] o>processing_type << $processing_type:string
{x}

// ============================================================================
// Variadic Set Intersection (<<< shorthand)
// ============================================================================

{|} |Set.Intersection
[<] i<type:pg.type
[<] i<sets:serial                      // Dynamic type (unknown at compile)
   [%] %InStream << #IO.Stream.TriplePull    // Enable <<<
   [%] %Keys << #IO.Stream.Keys.numeric      // Numeric indexing
   [%] %ItemType << $type                    // Type validation

[t] |T.Call
[W] |W.Polyglot.Scope

/*
 * Recursive Tournament Random Pairs
 * Takes collection and randomly pairs
 * from leafs to root in Tournament fashion
 * Recursively until we have one item
 */
[p] ~Recursive.Tournament.RandomPairs
   <array << $sets
   >item1 >> $set1
   >item2 >> $set2
   [r] $result << |Set.IntersectTwo
      <set1 << $set1
      <set2 << $set2
      >intersection >> $result
   [>] >winner << $result

[>] o>intersection << $result:set
{x}
```

## Syntax Summary

### Markers
- `{@}` - Registry definition
- `{|}` - Pipeline definition
- `{#}` - Enum/struct definition
- `{A}` - Alias definition block
- `{<>}` - Shorthand definition (future)
- `{x}` - Close block
- `[<]` - Input marker
- `[>]` - Output marker
- `[r]` - Runtime variable
- `[p]` - Parallel/procedure
- `[f]` - Conditional (yes)
- `[m]` - Match expression
- `[?]` - Match case
- `[t]` - Trigger
- `[W]` - Wrapper
- `[A]` - Alias/attribute
- `[.]` - Field
- `[+]` - Multi-line continuation
- `[%]` - Metadata
- `[v]` - Vacuum/aggregation (existing)
- `[z]` - Try (existing)

### Operators
- `$` - Variable prefix
- `:` - Type prefix
- `#` - Enum/struct prefix
- `|` - Pipeline prefix
- `!` - Error prefix
- `@` - Registry prefix
- `%` - Metadata prefix
- `.` - Hierarchy separator (universal)
- `<<` - Assignment/input
- `>>` - Output
- `<~` - Default value
- `i<` - Input parameter (definition)
- `o>` - Output parameter (definition)
- `<` - Input parameter (call)
- `>` - Output parameter (call)
- `<<<` - Variadic input (shorthand)
- `>>>` - Variadic output (shorthand)
- `?` - Conditional suffix
- `!?` - Negated conditional
- `=?` - Equality test
- `!=?` - Not equal test
- `>?`, `<?`, `>=?`, `<=?` - Comparison tests
- `?[min, max]` - Range (inclusive)
- `?(min, max)` - Range (exclusive)
- `in?` - Collection membership
- `!in?` - NOT in collection
- `re?` - Regex match
- `!re?` - Regex NOT match
- `|>` - Pipeline composition
- `~` - Unpack/iterate
- `*` - Wildcard
- `..` - Range (in string literals)
