# Polyglot v0.0.3.1 - Loop Unpack/Pack System Specification

**Version:** 0.0.3.1
**Date:** 2025-12-11
**Status:** ✅ APPROVED - Implementation Ready

---

## Overview

Version 0.0.3.1 introduces the **Loop Unpack/Pack System** - a revolutionary approach to iteration where each loop iteration is treated as a **mini-pipeline** with explicit I/O.

**Key Innovation:** Loops are not just control flow - they are composable mini-pipelines that unpack collections into iterations and pack results back into collections.

---

## Design Philosophy

> **"Each iteration is a mini-pipeline with named I/O."**

### Core Principles

1. **Iterations as Mini-Pipelines**
   - Each iteration has its own scope
   - Explicit inputs from main pipeline
   - Explicit outputs to main pipeline
   - Complete isolation between scopes

2. **Explicit Data Flow**
   - Unpack: Main scope → Iteration scope
   - Pack: Iteration scope → Main scope
   - No implicit state sharing
   - No mutable variables

3. **Composable Patterns**
   - Map, filter, reduce, chain
   - All expressed as pack/unpack operations
   - Uniform syntax across patterns

## Operator System

### Three Operator Types

| Marker | Operator | Purpose | Scope Transition |
|--------|----------|---------|------------------|
| `[|]` | Pipeline I/O | `<` input, `>` output | Pipeline definition/call |
| `[~]` | Unpack | `<` input, `>` output | Main → Iteration |
| `[*]` | Pack | `<` input, `>` output | Iteration → Main |

### Operator Consistency

All three operators use the same pattern:
- `<` for inputs (data flowing in)
- `>` for outputs (data flowing out)
- Marker determines the context

```polyglot
// Pipeline I/O
[r] |Pipeline
[|] <param << $var                    // Pipeline input
[|] >result >> $var                   // Pipeline output

// Unpack (spread)
[r] ~ForEach
[~] <array << $var                    // Unpack input: collection
[~] >item >> $var                     // Unpack output: iteration variable

// Pack (gather)
[V] *Join.All
[*] <item << $var                     // Pack input: from iteration
[*] >array >> $var                    // Pack output: to main
```

## Unpack Operator: `[~]`

### Purpose

**Unpack binds data from main pipeline scope into iteration mini-pipeline scope.**

### Syntax

```polyglot
[r/p/b] ~UnpackOperator               // Execution mode + operator
[~] <input_name << $main_var          // From main scope
[~] >output_name >> $iteration_var    // To iteration scope (each)
```

### Semantics

**`[~] <array << $items`**
- Takes `$items` from main pipeline scope
- Unpacks/iterates over the collection
- Feeds elements into iteration mini-pipelines

**`[~] >item >> $item`**
- For each iteration mini-pipeline
- Creates `$item` variable in iteration scope
- `$item` is isolated to that iteration

### Example: Basic ForEach

```polyglot
{|} |ProcessItems
[|] <items:array.int << $input_items

[t] |T.Call
[W] |W.Polyglot.Scope

[r] ~ForEach
[~] <array << $input_items            // Unpack: main scope variable
[~] >item >> $item                    // Unpack: iteration scope variable

   // Iteration mini-pipeline scope (indented 3 spaces)
   [r] $doubled << $item * 2

   [V] *Join.All
   [*] <item << $doubled
   [*] >array >> $results

[>] o>results << $results:array.int
{x}
```

### Multiple Unpack Outputs

Some unpack operators provide multiple iteration variables:

```polyglot
[r] ~Enumerate
[~] <array << $items                  // Input: collection
[~] >index >> $i                      // Output: 0, 1, 2, ...
[~] >item >> $item                    // Output: element at index

   [r] $indexed << #IndexedItem
      [.] .index << $i
      [.] .item << $item

   [V] *Join.All
   [*] <result << $indexed
   [*] >array >> $indexed_items
```

**Note:** Multiple outputs are provided by Polyglot standard library operators, not user-defined.

## Pack Operator: `[*]`

### Purpose

**Pack binds data from iteration mini-pipeline scope into main pipeline scope.**

### Syntax

```polyglot
[V] *PackOperator                     // Pack operator name
[*] <input_name << $iteration_var     // From iteration scope (each)
[*] >output_name >> $main_var         // To main scope (collected)
```

### Semantics

**`[*] <item << $result`**
- Collects variable `$result` from **each** iteration
- Aggregates across all iterations
- Behavior depends on pack operator

**`[*] >array >> $results`**
- Outputs collected results to main pipeline scope
- Creates `$results` variable in main scope
- Type depends on pack operator (array, map, single value, etc.)

### Example: Parallel Join

```polyglot
[p] ~ForEach
[~] <array << $urls
[~] >item >> $url

   [r] |HTTP.Get
   [|] <url << $url
   [|] >data >> $response

   [V] *Join.All                      // Wait for all parallel iterations
   [*] <item << $response             // Collect: each $response
   [*] >array >> $all_responses       // Output: array in main scope

[r] // Use $all_responses here
```

## Scope Boundaries

### Iteration Mini-Pipeline Scope

```polyglot
[r] ~ForEach
[~] <array << $items                  // ← Main pipeline scope
[~] >item >> $item                    // ← Boundary: main → iteration

   // ↓ Iteration mini-pipeline scope (3-space indentation)
   [r] $processed << |Transform <input << $item
   [r] $doubled << $processed * 2     // Only visible in iteration

   [V] *Join.All                      // ← Boundary marker
   [*] <item << $doubled              // ← Boundary: iteration → main
   [*] >array >> $results             // ← Main pipeline scope

[r] // Use $results here (main scope)
```

### Scope Rules

1. **Iteration scope starts:**
   - After `[~]` unpack markers
   - Marked by 3-space indentation

2. **Iteration scope ends:**
   - At `[V]` pack operator marker
   - Dedentation to loop level

3. **Variable visibility:**
   - Iteration variables NOT visible in main scope
   - Main variables NOT visible in iteration scope
   - Must explicitly pass via `[~]` unpack
   - Must explicitly collect via `[*]` pack

4. **Scope isolation:**
   - Each iteration has independent scope
   - No shared state between iterations
   - No mutable variables needed

## Execution Modes

### Sequential: `[r] ~*`

**Iterations execute one after another in order**

```polyglot
[r] ~ForEach
[~] <array << $items
[~] >item >> $item

   [r] $processed << |Transform <input << $item

   [V] *Join.All
   [*] <item << $processed
   [*] >array >> $results             // Order preserved
```

**Characteristics:**
- ✅ Order guaranteed
- ✅ Can depend on previous iterations (chaining)
- ❌ Slower for large datasets
- ✅ Predictable execution

### Parallel: `[p] ~*`

**Iterations execute concurrently, must specify join**

```polyglot
[p] ~ForEach
[~] <array << $items
[~] >item >> $item

   [r] $processed << |ExpensiveOp <input << $item

   [V] *Join.All                      // REQUIRED for parallel
   [*] <item << $processed
   [*] >array >> $results             // Order NOT guaranteed
```

**Characteristics:**
- ❌ Order NOT guaranteed (unless pack operator specifies)
- ❌ Cannot chain iterations
- ✅ Faster for independent operations
- ✅ Must have `[V]` pack operator

### Fire-and-Forget: `[b] ~*`

**Iterations execute without waiting for results**

```polyglot
[b] ~ForEach
[~] <array << $events
[~] >item >> $event

   [r] |LogEvent <event << $event
   [r] |SendWebhook <event << $event

// NO [V] or [*] - fire-and-forget has no collection
```

**Characteristics:**
- ❌ No results collected
- ✅ Non-blocking
- ✅ Useful for side effects (logging, notifications)
- ❌ No pack operator (`[V]` / `[*]` omitted)

## Standard Unpack Operators

### `~ForEach` (Standard Iteration)

**Iterate over array elements**

```polyglot
[r] ~ForEach
[~] <array << $items                  // Input: array
[~] >item >> $item                    // Output: each element
```

### `~Enumerate` (With Index)

**Iterate with index and element**

```polyglot
[r] ~Enumerate
[~] <array << $items
[~] >index >> $i                      // Output: 0, 1, 2, ...
[~] >item >> $item                    // Output: element
```

### `~Range` (Numeric Iteration)

**Generate numeric sequence**

```polyglot
[r] ~Range
[~] <start << 0
[~] <end << 100
[~] <step << 5
[~] >index >> $i                      // Output: 0, 5, 10, 15, ..., 95
```

### `~Zip` (Parallel Arrays)

**Iterate over two arrays in parallel**

```polyglot
[r] ~Zip
[~] <array1 << $names
[~] <array2 << $ages
[~] >item1 >> $name
[~] >item2 >> $age
```

### `~Window` (Sliding Window)

**Iterate with sliding window**

```polyglot
[r] ~Window
[~] <array << $prices                 // [10,20,30,40,50]
[~] <size << 3
[~] <step << 1
[~] >window >> $window                // [10,20,30], [20,30,40], [30,40,50]
```

### `~Chunk` (Fixed Batches)

**Split into fixed-size chunks**

```polyglot
[r] ~Chunk
[~] <array << $items                  // [1,2,3,4,5,6,7,8]
[~] <size << 3
[~] >chunk >> $batch                  // [1,2,3], [4,5,6], [7,8]
```

### `~ForEach.Chained` (Transformation Chain)

**Chain iteration outputs as next inputs**

```polyglot
[r] ~ForEach.Chained
[~] <array << $transforms
[~] <seed << $initial                 // Initial value
[~] >item >> $transform               // Current transform
[~] >current >> $value                // Chained value
```

### `~Reduce` (Accumulator)

**Maintain accumulator across iterations**

```polyglot
[r] ~Reduce
[~] <array << $numbers
[~] <initial << 0
[~] >item >> $num
[~] >accumulator >> $acc
```

### `~While` / `~Until` (Conditional Loops)

**Loop while/until condition**

```polyglot
[r] ~While
[~] <condition << |HasMore
[~] >item >> $current

[r] ~Until
[~] <condition << |IsDone
[~] >item >> $current
```

## Standard Pack Operators

### `[V] *Join.All` (Collect All Results)

**Wait for all iterations, collect all results**

```polyglot
[V] *Join.All
[*] <item << $result
[*] >array >> $results                // Array of all results
```

**Behavior:**
- Blocks until all iterations complete
- Collects all values
- Sequential: order preserved
- Parallel: order unspecified

### `[V] *Join.First` (Race Pattern)

**Return first to complete, cancel others**

```polyglot
[V] *Join.First
[*] <item << $result
[*] >value >> $first_result           // Single value (first)
```

**Behavior:**
- Returns as soon as first iteration completes
- Cancels remaining iterations
- Only works with `[p]` parallel

### `[V] *Join.Any` (First Success)

**Return first successful iteration**

```polyglot
[V] *Join.Any
[*] <item << $result
[*] >value >> $first_success          // First non-error result
```

**Behavior:**
- Returns first iteration without error
- Ignores failed iterations
- Continues until success found

### `[V] *Join.All.Success` (Only Successful)

**Collect only successful iterations**

```polyglot
[V] *Join.All.Success
[*] <item << $result
[*] >array >> $successes              // Array of non-failed results
```

**Behavior:**
- Waits for all iterations
- Only collects iterations in `Ready` state
- Ignores `Failed` state iterations

### `[V] *Join.All.Failures` (Only Failures)

**Collect only failed iterations**

```polyglot
[V] *Join.All.Failures
[*] <item << $result
[*] >array >> $failures               // Array of error information
```

**Behavior:**
- Waits for all iterations
- Only collects iterations in `Failed` state
- Contains error information

### `[V] *Chain` (Feed to Next Iteration)

**Chain iteration output as next iteration input**

```polyglot
[V] *Chain
[*] <current << $result
[*] >next >> >item                    // Feeds to next iteration's [~] >item
```

**Behavior:**
- Each iteration's output → next iteration's input
- Only works with `[r]` sequential
- Creates transformation pipeline

### `[V] *CollectLast` (Only Last Iteration)

**Collect only from final iteration**

```polyglot
[V] *CollectLast
[*] <last << $result
[*] >value >> $final_result           // Single value (last only)
```

**Behavior:**
- Only collects from final iteration
- All previous iterations discarded
- Output: single value

### `[V] *Reduce` (Accumulator Pattern)

**Maintain accumulator across iterations**

```polyglot
[V] *Reduce
[*] <accumulator << $new_acc
[*] >value >> $final_accumulator      // Single accumulated value
```

**Behavior:**
- Updates accumulator each iteration
- Final accumulator value output
- Only works with `[r]` sequential

## Complete Examples

### Example 1: Parallel HTTP Requests

```polyglot
{|} |FetchMultipleURLs
[|] <urls:array.string << $url_list

[t] |T.Call
[W] |W.Polyglot.Scope

[p] ~ForEach
[~] <array << $url_list
[~] >item >> $url

   [r] |HTTP.Get
   [|] <url << $url
   [|] >data >> $response
   [|] >error >> $http_error

   [y] $http_error;state =? #;pg;state;failed
      [r] $result << "FAILED"
   [y] *?
      [r] $result << $response

   [V] *Join.All
   [*] <item << $result
   [*] >array >> $all_responses

[>] o>responses << $all_responses:array.string
{x}
```

### Example 2: Chained Data Transformations

```polyglot
{|} |TransformData
[|] <initial:serial << $raw_data
[|] <steps:array.pipeline << $transform_steps

[t] |T.Call
[W] |W.Polyglot.Scope

[r] ~ForEach.Chained
[~] <array << $transform_steps
[~] <seed << $initial
[~] >item >> $transform
[~] >current >> $value

   [r] $transform
   [|] <input << $value
   [|] >output >> $next_value

   [V] *Chain
   [*] <current << $next_value
   [*] >next >> >current              // Chain to next

   [V] *CollectLast
   [*] <last << $next_value
   [*] >main >> $final_result

[>] o>result << $final_result:serial
{x}
```

### Example 3: Reduce Pattern (Sum)

```polyglot
{|} |SumNumbers
[|] <numbers:array.int << $number_list

[t] |T.Call
[W] |W.Polyglot.Scope

[r] ~Reduce
[~] <array << $number_list
[~] <initial << 0
[~] >item >> $num
[~] >accumulator >> $acc

   [r] $new_acc << $acc + $num

   [V] *Reduce
   [*] <accumulator << $new_acc
   [*] >value >> $total

[>] o>sum << $total:int
{x}
```

### Example 4: Separate Success/Failure Collections

```polyglot
{|} |ProcessWithErrors
[|] <items:array.serial << $input_items

[t] |T.Call
[W] |W.Polyglot.Scope

[p] ~ForEach
[~] <array << $input_items
[~] >item >> $item

   [r] |MayFail
   [|] <input << $item
   [|] >result >> $processed

   [V] *Join.All.Success
   [*] <success << $processed
   [*] >array >> $successes

   [V] *Join.All.Failures
   [*] <failure << $processed
   [*] >array >> $failures

[>] o>successes << $successes:array.serial
[>] o>failures << $failures:array.error
{x}
```

### Example 5: Fire-and-Forget Notifications

```polyglot
{|} |SendNotifications
[|] <events:array.Event << $event_list

[t] |T.Call
[W] |W.Polyglot.Scope

[b] ~ForEach
[~] <array << $event_list
[~] >item >> $event

   [r] |LogToDatabase
   [|] <event << $event

   [r] |SendEmail
   [|] <event << $event

   [r] |SendWebhook
   [|] <event << $event

// No [V] or [*] - fire-and-forget
{x}
```

## Variable States and Error Handling

### Variable States in Polyglot

All variables have states:
- `Ready` - Has valid value
- `Failed` - Has error information
- `Pending` - Waiting for value
- `Default` - Using default value

### Checking Variable State

```polyglot
[r] |MayFail
[|] <input << $item
[|] >result >> $processed

// Check state
[y] $processed;state =? #;pg;state;ready
   [r] // Use $processed
[y] $processed;state =? #;pg;state;failed
   [r] // Handle error
```

### Error Handling in Loops

**Strategy 1: Handle errors in iteration**

```polyglot
[p] ~ForEach
[~] <array << $items
[~] >item >> $item

   [r] |MayFail
   [|] <input << $item
   [|] >result >> $processed

   [y] $processed;state =? #;pg;state;failed
      [r] $safe_value << "ERROR"
   [y] *?
      [r] $safe_value << $processed

   [V] *Join.All
   [*] <item << $safe_value
   [*] >array >> $results
```

**Strategy 2: Separate success/failure collections**

```polyglot
[p] ~ForEach
[~] <array << $items
[~] >item >> $item

   [r] |MayFail
   [|] <input << $item
   [|] >result >> $processed

   [V] *Join.All.Success
   [*] <success << $processed
   [*] >array >> $successes

   [V] *Join.All.Failures
   [*] <failure << $processed
   [*] >array >> $failures
```

## Named Pack Parameters

### Problem: Multiple Pack Operations

When collecting multiple different values, ambiguity can arise:

```polyglot
[V] *Partition
[*] <item << $result                  // ❓ Which output?
[*] <item << $error                   // ❓ Which output?
[*] >array >> $results
[*] >array >> $errors
```

### Solution: Named Pack Parameters

**Use distinct names for inputs and outputs:**

```polyglot
[V] *Partition
[*] <success << $result               // Named: success
[*] <failure << $error                // Named: failure
[*] >successes >> $results            // Output for success
[*] >failures >> $errors              // Output for failure
```

**Mapping rule:** Pack operator matches by name
- `<success` → `>successes`
- `<failure` → `>failures`

## Grammar Summary

### Loop Structure

```ebnf
loop ::= execution_mode unpack_operator
         unpack_inputs
         unpack_outputs
         iteration_body
         pack_operator
         pack_inputs
         pack_outputs

execution_mode ::= '[r]' | '[p]' | '[b]'

unpack_operator ::= '~' identifier ('.' identifier)*

unpack_inputs ::= ('[~]' '<' identifier '<<' variable)+

unpack_outputs ::= ('[~]' '>' identifier '>>' variable)+

iteration_body ::= (indented_statement)+

pack_operator ::= '[V]' '*' identifier ('.' identifier)*

pack_inputs ::= ('[*]' '<' identifier '<<' variable)+

pack_outputs ::= ('[*]' '>' identifier '>>' variable)+
```

### Complete Example Structure

```polyglot
[r/p/b] ~OperatorName
[~] <input_name << $main_var
[~] >output_name >> $iteration_var

   // Iteration mini-pipeline (3-space indent)
   [r] // ... statements ...

   [V] *PackOperator
   [*] <input_name << $iteration_var
   [*] >output_name >> $main_var
```

## Implementation Notes

### Parser Requirements

1. **New markers:** `[~]`, `[*]`, `[V]`
2. **Scope tracking:** Indentation-based iteration scope
3. **Operator patterns:** `~Operator` and `*Operator`
4. **State checking:** `$var;state` syntax

### Compiler Requirements

1. **Scope isolation:** Generate isolated iteration scopes
2. **Pack/unpack binding:** Map variables between scopes
3. **Execution mode:** Generate sequential/parallel/fire-and-forget code
4. **Join semantics:** Implement pack operator behaviors

### Runtime Requirements

1. **Variable states:** Track Ready/Failed/Pending/Default states
2. **Parallel execution:** Thread pool or async runtime
3. **Join operations:** Wait strategies for pack operators
4. **Error propagation:** Handle Failed state across iterations
