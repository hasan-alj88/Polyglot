# Polyglot v0.0.4 - Universal Alias System

**Date:** 2025-12-11
**Status:** ✅ APPROVED - Design Complete

---

## The Problem

Polyglot uses hierarchies extensively for organization:
```polyglot
:pg.array.pg.string                    // Verbose but clear
#Boolean.True                          // Verbose for common values
|Database.Users.FindByEmail            // Long but organized
!Validation.Schema.InvalidJSON         // Hierarchical error taxonomy
```

**Trade-off:** Clarity vs. Verbosity

**Solution:** Explicit alias system that lets you choose.

---

## The Design: `{A}` Alias Block

### New Block Type: `{A} ... {x}`

```polyglot
{A}                                    // Alias declaration block
[A] source >> alias                    // Export/map operator
[A] alias << source                    // Import/map operator
{x}                                    // Close block
```

**Operators:**
- `>>` - "maps to" / "exports as" (source to alias)
- `<<` - "maps from" / "imports as" (alias from source)

Both are equivalent, use whichever reads better!

```polyglot
{@} @Polyglot::Types:0.0.0.1

{A}

[A] :pg.error >> :error

// Collections
[A] :pg.array >> :array

// Boolean values (common case!)
[A] #Boolean.True >> #True
[A] #Boolean.False >> #False
{x}
```

## Using Aliases in Your Code

### Import Aliases via Registry Reference

```polyglot
{@} @Local::MyApp:1.0.0.0
[A] @App
[<] @PgTypes                           // Import all aliases from @PgTypes
{x}

{|} |Pipeline.ProcessUser
[<] ,user_id:string                    // Uses alias (:pg.string)
[<] ,age:int                           // Uses alias (:pg.int)
[<] ,tags:array.string                 // Composed: :pg.array.pg.string
[<] ,metadata:serial                   // Uses alias (:pg.serial)

[t] |T.Call
[W] |W.Polyglot.Scope

// Boolean aliases in action
[r] ,found:bool << #False              // Instead of #Boolean.False
[r] ,is_valid:bool << #True            // Instead of #Boolean.True

// Mix aliased and full forms
[r] ,errors:array.error << :array.error
[r] ,backup:pg.array.pg.error << :pg.array.pg.error  // Full form still works!

[y] ,found =? #False
   [r] ,found << #True                 // Clean!

[>] ,found:bool
{x}
```

### Without Importing Aliases

```polyglot
{@} @Local::ExplicitApp:1.0.0.0
[A] @Explicit
// No [<] @PgTypes import!
{x}

{|} |Pipeline.ProcessUser
[<] ,user_id:pg.string                 // Must use full names
[<] ,age:pg.int
[<] ,found:pg.bool

[t] |T.Call
[W] |W.Polyglot.Scope

[r] ,found << #Boolean.False           // Full enum path required

[>] ,found:pg.bool
{x}
```

**Both styles work!** Choose based on preference/project guidelines.

## Creating Custom Aliases

### Project-Specific Aliases

```polyglot
{@} @MyCompany::ProjectAliases:1.0.0.0
[A] @MyAliases
{x}

{A}
// Type aliases
[A] :MyApp.UserRecord >> :User
[A] :MyApp.OrderRecord >> :Order
[A] :Database.Schema.V2 >> :Schema

// Pipeline aliases (shorten common operations!)
[A] |Database.Users.FindByEmail >> |FindUser
[A] |Database.Orders.FindById >> |FindOrder
[A] |Validation.Email.Check >> |ValidateEmail

// Error aliases (shorter error names!)
[A] !Validation.Email.InvalidFormat >> !InvalidEmail
[A] !Database.Connection.Timeout >> !DBTimeout
[A] !Network.HTTP.NotFound >> !NotFound

// Enum aliases
[A] #OrderStatus.Processing >> #Processing
[A] #OrderStatus.Shipped >> #Shipped
{x}

// Use in other files:
{@} @Local::OrderService:1.0.0.0
[A] @OrderSvc
[<] @MyAliases                         // Import project aliases
{x}

{|} |Pipeline.ProcessOrder
[<] ,order:Order                       // Instead of :MyApp.OrderRecord
[<] ,user:User                         // Instead of :MyApp.UserRecord

[t] |T.Call
[W] |W.Polyglot.Scope

// Aliased pipeline call!
[r] |FindOrder                         // Instead of |Database.Orders.FindById
   <order_id:string << ,order.id
   >order:serial >> ,order_data

[r] ,status << #Processing             // Instead of #OrderStatus.Processing

[y] ,order_data =? #False
   [r] ~!NotFound                      // Instead of !Network.HTTP.NotFound
      <message:string << "Order not found"

[>] ,status:OrderStatus
{x}
```

## Universal Alias System

### Works for ALL Prefix Types

| Prefix | Full Form | Alias Form | Alias Declaration |
|--------|-----------|------------|-------------------|
| `:` Type | `:pg.array.pg.string` | `:array.string` | `[A] :pg.array >> :array` |
| `#` Enum | `#Boolean.True` | `#True` | `[A] #Boolean.True >> #True` |
| `\|` Pipeline | `\|Database.Users.Find` | `\|FindUser` | `[A] \|Database.Users.Find >> \|FindUser` |
| `!` Error | `!Network.HTTP.Timeout` | `!Timeout` | `[A] !Network.HTTP.Timeout >> !Timeout` |
| `,` Variable | (no aliasing needed) | - | - |

## Alias Composition

### Aliases Compose in Hierarchies

```polyglot
{A}
[A] :pg.array >> :array
[A] :pg.string >> :string
[A] :pg.error >> :error
{x}

// With @PgTypes imported:
[r] ,tags:array.string                 // :pg.array + .pg.string = :pg.array.pg.string
[r] ,errors:array.error                // :pg.array + .pg.error = :pg.array.pg.error
[r] ,data:array.serial                 // :pg.array + .pg.serial = :pg.array.pg.serial
```

**Resolution rule:** Aliases expand before hierarchy navigation.

## Compiler Responsibilities

### 1. Namespace Collision Detection

```polyglot
{A}
[A] :pg.int >> :int                    ✅
[A] :MyLib.int >> :int                 ❌ ERROR: :int already aliased!
{x}
```

**Error:**
```
Alias collision: :int
  First defined:  :pg.int >> :int
  Redefined as:   :MyLib.int >> :int
  Resolution: Use different alias name or don't import conflicting registries
```

### 2. Undefined Alias Detection

```polyglot
// Without @PgTypes import:
[r] ,count:int << 0                    ❌ ERROR: :int is not defined
```

**Error:**
```
Type :int not found
  Did you mean: :pg.int?
  Hint: Import @PgTypes registry: [<] @PgTypes
```

### 3. Full Form Always Works

```polyglot
// Even with @PgTypes imported:
[r] ,count:pg.int << 0                 ✅ Always valid
[r] ,count:int << 0                    ✅ Valid if @PgTypes imported
```

**Both forms resolve to the same type.**

## Benefits

### 1. Solves Verbosity for Deep Hierarchies

**Before:**
```polyglot
[r] |Database.Connection.Pool.Acquire
   <config:Database.Config.ConnectionSettings << ,db_config
   >connection:Database.Connection.Handle >> ,conn

[y] ,conn =? #Database.Connection.Status.Timeout
   [r] ~!Database.Connection.Error.PoolExhausted
```

**After (with aliases):**
```polyglot
{A}
[A] |Database.Connection.Pool.Acquire >> |AcquireConn
[A] :Database.Config.ConnectionSettings >> :DBConfig
[A] :Database.Connection.Handle >> :DBHandle
[A] #Database.Connection.Status.Timeout >> #Timeout
[A] !Database.Connection.Error.PoolExhausted >> !PoolExhausted
{x}

[r] |AcquireConn
   <config:DBConfig << ,db_config
   >connection:DBHandle >> ,conn

[y] ,conn =? #Timeout
   [r] ~!PoolExhausted
```

**~60% fewer characters, dramatically more readable!**

### 2. Explicit and Greppable

```bash
# Find alias definitions
grep ">> :User"              # Find what aliases to :User

# Find alias imports
grep "[<] @PgTypes"          # Find files using standard aliases

# Full forms still greppable
grep ":pg.int"               # Find explicit uses
grep ">> :int"               # Find alias definition
```

### 3. Team Flexibility

**Explicit team:** Don't import aliases, use full names
```polyglot
{@} @Local::ExplicitService:1.0.0.0
[A] @Svc
// No alias imports
{x}
```

**Concise team:** Import common aliases
```polyglot
{@} @Local::ConciseService:1.0.0.0
[A] @Svc
[<] @PgTypes
[<] @CompanyAliases
{x}
```

Both work! No forcing one style.

### 4. Migration Path

```polyglot
// Phase 1: Use full names
[r] ,count:pg.int << 0

// Phase 2: Create aliases (backward compatible!)
{A}
[A] :pg.int >> :int
{x}

// Phase 3: Gradually adopt (both work!)
[r] ,count:pg.int << 0         // Old code still works
[r] ,total:int << 0            // New code uses alias
```

No breaking changes!

## Syntax Summary

### Alias Declaration Block

```polyglot
{A}                                    // Open alias block
[A] source >> alias                    // Map source to alias
[A] alias << source                    // Map alias from source (equivalent)
{x}                                    // Close alias block
```

### Alias Registry

```polyglot
{@} @Namespace::Name:Version
[A] @RegistryAlias                     // Registry short name
{x}

{A}
[A] :long.type.name >> :short
{x}
```

### Importing Aliases

```polyglot
{@} @Local::MyApp:1.0.0.0
[A] @App
[<] @PgTypes                           // Import alias registry
[<] @CompanyAliases                    // Can import multiple!
{x}
```

### Using Aliases

```polyglot
// With import:
[r] ,count:int                         // Aliased form
[r] ,count:pg.int                      // Full form (still works!)

// Without import:
[r] ,count:pg.int                      // Must use full form
```

## Rules

1. ✅ **Alias declarations in `{A}` blocks only**
2. ✅ **Import aliases via `[<] @Registry` in `{@}` block**
3. ✅ **Both aliased and full forms always work**
4. ✅ **Compiler detects namespace collisions**
5. ✅ **Aliases compose in hierarchies** (`:array.string` works)
6. ✅ **Works for all prefix types** (`:`, `#`, `|`, `!`)
7. ✅ **Aliases are compile-time** (zero runtime overhead)

---

## Design Philosophy

**"Polyglot uses hierarchies extensively for organization. Aliases let you choose clarity vs. brevity on a per-file basis."**

- **Hierarchies** provide organization and namespace
- **Aliases** provide ergonomics and readability
- **Explicit imports** provide clarity about what's being used
- **Full forms always work** so no lock-in

**Universal hierarchy pattern + explicit aliasing = Best of both worlds!**
